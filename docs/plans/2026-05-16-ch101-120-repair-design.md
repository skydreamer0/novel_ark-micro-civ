# 第101～120章混稿修復執行設計

> 建立日期：2026-05-16
> 依據：`docs/roadmap/2026-05-16-ch101-120-cleanup-roadmap.md`、`docs/roadmap/2026-05-16-ch101-120-audit-table.md`
> 範圍：第四卷《投放名冊》第101～120章

## 問題摘要

第101-120章已確診四類系統性污染：
1. 流程殘留進正文（`工作流：Write Chapter` 等）
2. 模板句大量重複（同一句連續重複數十次）
3. 企劃語混入正文（「本章真正的爽點是……」）
4. 章節功能互相覆蓋（111-120章共享同一組骨架）

## 執行策略

三個子代理平行作業：

| 代理 | 範圍 | 工作內容 | 檔案 |
|------|------|----------|------|
| Agent 1 | Phase 2 | 清理第101-110章表層污染 | 10個獨立md檔案 |
| Agent 2 | Phase 3A | 重寫第111-115章 | 5個獨立md檔案 |
| Agent 3 | Phase 3B | 重寫第116-120章 | 5個獨立md檔案 |

完成後接續：
- Phase 4：同步資料庫狀態（目前進度.md、伏筆追蹤.md、第120章錨點）
- Phase 5：建立續寫防混規則

## 修復原則

1. 第101-110章：保留主事件，刪除污染頭尾與通用治理段
2. 第111-120章：保留事件錨點，逐章重寫場景核心
3. 每章3000-4500字繁體中文
4. 符合 `02_PROJECT_DATABASE/00_寫作規則.md` 所有規範
5. 第120章完成後必須可作為卷末狀態錨點

## 成功標準

- 每章正文不含 `工作流：`、`模式B交付摘要`、企劃語
- 單章內無重複句
- 每章有獨立事件、場景、情緒轉折
- 第120章可作為第五卷開局錨點
