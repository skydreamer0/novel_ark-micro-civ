# 第96章 財報結算與民生鏡頭

> 章節規劃版（依 chapter-planner.md）

## 1. 本章一句話核心
第十八段競價系統雖然被撕開缺口，卻立刻讓大量底層人湧向安全通道並觸發高層「局部消除」預警，林燼第一次清楚看見自己的勝利也能變成別人的死刑。

## 2. 四段式節奏規劃
1. 開局直接接在伺服器失穩後的外部反應，競價門禁短暫鬆開，底層拾荒者與病患像潰堤一樣往安全通道衝，林燼還來不及慶祝，就得先處理秩序崩盤的現場。
2. 隊伍一邊收攏戰利品、一邊切斷上游追蹤，主機同步拆解伺服器殘骸與壟斷算法，完成「投入 -> 轉化 -> 產出」的第一輪財報結算。
3. 當蘇岚把競價核心殘留模型拼起來，眾人發現平民湧入安全通道本身就是一個會觸發高層清洗閾值的參數，剛才那場看似救人的行動，正在把更多人送進下一輪消除名單。
4. 民生鏡頭落到具體人群：剛拿到過站機會的家庭、急著送病患過關的搬運工、剛脫離高熵裂縫的人，全都在不知情中再次被高層標價，為第97章的內部倫理衝突蓄壓。

## 3. 本章推動的主線
- 真相線：戰爭需求模型第一次以「參數反應」的形式證明，高層會根據底層流動與損耗即時調整清洗。
- 戰力線：主機吸收壟斷算法碎片，為後續規則偽裝與路由追蹤提供基礎。
- 文明線：外部救援行動的副作用，迫使內部文明與隊伍開始面對「勝利不等於善果」的治理問題。

## 4. 本章主要衝突與轉折
主要衝突不是能不能搶到物資，而是要不要承認這次破壞系統可能反而害了更多人。真正的轉折是模型把「平民湧入」直接標成消除觸發因子，讓林燼無法再把後果外包給高層冷血。

## 5. 本章代價與收穫
代價是隊伍背上更沉重的道德帳與更高的追索風險；收穫則是稅金、裝備物資與壟斷算法碎片到手，讓行動不是毫無產出。

## 6. 本章不可逆變化
第十八段的局部消除預警一旦被觸發，就不可能再回到原本的低烈度剝削狀態；同時，林燼也正式踩進「自己的選擇會放大他人死亡」的統御難題。

## 7. 本章章尾鉤子
模型殘片裡開始浮出一份更大清單的索引提示，為第97章的「投放名冊」概念埋下直指性入口。

## 8. 設定衝突檢查
本章符合第四卷財務節點要求，把資源收穫與民生代價同時寫出，不讓制度壓迫只停留在抽象名詞。沒有新增新技術，只延續第95章拿到的系統殘骸與模型痕跡。

## 9. 可能產生拼接感的位置與修補方案
這章容易寫成戰後流水帳。修補方式是讓民生潰堤、模型預警與戰利品拆解同步發生，讓「結算」本身就是壓力場，而不是降速空轉。
