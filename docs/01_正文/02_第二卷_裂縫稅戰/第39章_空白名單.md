# 第39章：空白名單

追查令下來後，灰鏈收到第一份「空白名單」。

那是一張沒有姓名、只有編號與年齡的表，從托養池內部匿名流出。名單上的孩子全被標註為「可加速分流」，時間集中在未來兩週。換言之，顧衡或其上游正準備用一波加速分流，對沖托養池事件帶來的輿論壓力。

用制度報復制度。

燕禾盯著那份名單，聲音發乾：「我們救出一個，他們就想讓十個更快進工序。」

林燼把名單投影放大，發現編號末段都帶同一個前綴：`S-Delta`。這與 `QW-Delta-17` 再次重疊。看來「腳本選樣」與「人口分流」是同一套上游邏輯：先挑可耗人群，再用稅與事故把周邊資源抽乾。

這次不能只救單人。

燼站開會三小時，最後做出一個極具風險的計畫：公開空白名單，但只公開一半。

公開太少，壓不動；公開太多，來源會被反查，名單內的人反而更快被清理。燕九設計了「半透明披露」：釋出編號規律、年齡區間與加速時間窗，不放完整對應，讓社區先自查、先保護，再由灰鏈暗線補齊重點個案。

蘇岚皺眉：「聽起來像在賭人心會合作。」

林燼點頭：「這一章本來就只能賭人心。」

消息在夜裡十一點投放到三個公共節點。不到一小時，回應超過預期：

- 有家屬主動提供孩子編號片段。
- 有醫療站願意免費做分流前健康證明（可延後流程）。
- 兩支影子配額隊伍提出拿一次入口窗口換灰鏈協助保人。

顧衡也在同夜反擊。他下令把「編號資訊外洩」定義為公共安全事件，授權區域代理就地扣押可疑終端。第九段街頭一夜多了無數臨檢點。

灰鏈節點壓力瞬間爆表。

燕九幾乎整夜沒離開鍵盤，持續分發動態路由，讓名單相關訊息在五分鐘內自毀。燕禾則帶著小岑和幾名志工，把確認到的高危編號做「雙層偽裝」：一層改變外觀資料，一層改變醫療優先級標記。

「我們像在給活人打補丁。」燕禾說。

「補丁總比判死好。」蘇岚回。

凌晨兩點，第一個重大危機爆發。西側社區的一名志工被抓，對方手機裡留有部分名單映射。若映射落到代理手上，至少十幾名孩子會在天亮前被轉移。

林燼立刻帶隊去截押送車。

這不是為了救一個志工，而是為了搶回那份映射。押送路線短、警戒高，正面硬打等於宣告全面開戰。林燼改用「交通癱瘓」策略：讓兩輛報廢搬運車在匝道口同時故障，拖慢押送車，再由燕九遠端發送「上游緊急徵調」假令，要求押送車臨時改道至檢核點。

押送隊果然中計，進入一段無監控維修廊。蘇岚破門、林燼控人、燕禾抹除終端，整段行動一分四十秒完成。志工救回，映射回收。

志工被救後一直道歉，說自己不夠小心。林燼只回一句：「你不是錯在被抓，你錯在以為自己只能一個人扛。」

天亮，空白名單事件還在發酵，但顧衡沒拿到完整映射。灰鏈趁這段空檔，把首批十二名高危孩子成功轉入「醫療觀察」流程，暫時避開加速分流。

十二不是很多，卻是第一次把「制度裡被消失的人」硬拉回可見範圍。

尾聲，唐雀帶來一條新消息：R-17 裂縫入口下週將改為「軍管試營運」，所有民間配額暫停。

這意味著，燼站若不在七天內拿到入口局部控制權，灰鏈的高值來源將被徹底掐斷。

林燼把空白名單收進防火盒，抬頭對眾人說：「先守人的階段結束。接下來，開始清場。」
