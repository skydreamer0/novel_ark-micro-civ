# 第37章：十二分鐘改模

十二分鐘有多長？

在平常，它夠煮一鍋壞掉的合成麵；在稅戰裡，它可能夠救六支隊、也夠讓一整條灰鏈暴露。

卓汐給的預簽鑰匙啟用時間是凌晨 02:14 到 02:26。這段窗口固定寫在上游更新排程裡，無法延長、無法重來。

燕九把任務切成四段：

1. 進入模型維護通道。
2. 定位兩個致命權重欄位。
3. 注入隨機噪音與延遲保護。
4. 清除改動痕跡，留下「更新抖動」假象。

每一段都可能失敗。

為降低風險，林燼決定採「雙層掩護」：外層由蘇岚在西工坊製造一次小規模配額衝突，把巡防指揮注意力拉向地面；內層由燕禾在社區端同時發出三條假申訴流量，壓住資料中繼負載，讓模型維護通道看起來只是例行卡頓。

真正的刀在燕九手上，真正的鍋在林燼背上。

02:12，兩人進入北弧館地下維護間。

02:14，陶瓷鑰匙插入，通道開。

終端跳出灰白介面，標題是：`事故傾向權重面板（僅限觀察）`。燕九罵了一句：「它還是只給觀察。」

林燼把卓汐留下的二段校驗碼輸入隱藏欄位，介面邊角出現一個幾乎看不見的黑點。燕九點下去，權限切換成功。

02:16，第一權重定位：`入口歷史違規係數`。

這個欄位被設成極端高值，導致只要某隊過去有一次異常，就會被無限放大風險，幾乎注定被判高事故。

02:18，第二權重定位：`代理回報可信加權`。

顧衡的代理系統回報在模型裡被給了超常可信度，意味著代理一句「高風險」，就能把一隊直接推進事故名單。

「真乾淨，」燕九冷笑，「把偏見寫成公式，再叫它科學。」

02:19，開始注入噪音。

燕九把兩欄位改成區間隨機，並綁上來源校驗延遲。簡單說，模型短期內仍會跑，但無法再穩定指定「誰先死」。

02:21，外層掩護出現波動。

蘇岚在頻道裡低聲：「巡防沒全去西工坊，留了兩組回防北弧。你們剩不到四分鐘。」

02:22，清痕程序卡住。

系統突然要求上游二次簽章。這一步卓汐沒提到，代表有人臨時加了保險。若拿不到簽章，改動會被標成惡意入侵。

燕九額角冒汗：「我可以硬刪日志，但會留下空洞。」

林燼盯著倒數，做了最危險的選擇：「不刪，偽造。」

他把過去三天社區申訴流量包的一部分噪音塞進更新日志，偽裝成「民生高峰導致模型漂移」，並附上自動恢復建議。這種作法不完美，但比硬刪更像真實故障。

02:25，日志偽造完成。

02:26，窗口關閉。

兩人剛拔線，門外就傳來腳步。蘇岚在外圍引爆預埋聲源，把追兵引去東側樓梯。林燼與燕九從通風夾層爬出，膝蓋在鐵網上磨出兩道血痕。

撤離成功，但真正結果要等天亮。

上午九點，第一批配額事故名單發布。

全段都愣住了——名單出現前所未有的分散，沒有任何一支隊伍被連續三次標記，代理勢力提交的高風險回報有一半被打回「待核驗」。

顧衡系統第一次出現「殺不準」。

唐雀看到名單時，笑到咳嗽：「你們真的把那把刀磨鈍了。」

但代價很快到來。中午十二點，審計塔發出新令：

> 「為確保戰時模型可靠，啟動人工覆核隊。」

這代表顧衡將把演算法失靈造成的缺口，改由真人補刀。模型可騙，人心更難騙。

林燼把公告關掉，轉向燕禾：「托養池那條線，進度？」

燕禾深吸一口氣：「卓汐女兒排在後天分流評估。我們只剩四十八小時。」

尾聲，行動板寫下下一節點：

`38：托養池撤離。`

灰鏈與稅戰的對抗，從改公式轉向搶人。
