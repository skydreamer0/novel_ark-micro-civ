# 小說資料庫 (Novel Database) 指引

本目錄是小說架構的「單一事實來源」(Single Source of Truth)，採用模組化結構以確保長篇連載的核心設定、邏輯與伏筆不發生衝突。

## 📁 結構說明

- **`01_worldview/`**：世界觀基礎。定義母艦、物理規則（熵潮）、歷史真相與揭密藍圖。
- **`02_system/`**：力量與規則系統。文明主機成長樹、資源轉化率與社會生存方程式。
- **`03_factions/`**：勢力與陣營。各勢力的壟斷資源、目標與仇恨鏈。
- **`04_characters/`**：角色檔案。包含「不可 OOC 原則」、人物轉折與價值對抗。
- **`05_plotlines/`**：主線與結構。全書總架構、核心衝突循環。
- **`06_volumes/`**：分卷大綱。各卷的主題、衝突與局勢演變細節。
- **`07_chapters/`**：章節開發。章節任務模板、細綱與追蹤紀錄。
- **`08_foreshadowing/`**：伏筆與辭典。名詞定義、伏筆埋設位置與回收狀態。

## 📏 維護規則
1. **編號排序**：所有文件使用 `XX_name.md` 命名，由 macro 到 micro 排序。
2. **待補標記**：若資訊不足，請明確標記為 `[待補]`。
3. **衝突標記**：發現新舊設定打架時，請標記 `[衝突]` 並在審計階段解決。
4. **禁止散文**：資料庫內容應以條列、表格或清單為主，方便 AI 提取與檢索。
