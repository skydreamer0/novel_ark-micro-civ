# 降維打臉五步法

> 本書所有爽點場景的結構模板。禁止在正文中出現這些標籤名稱。

---

## Step 1：規則壓迫

敵人使用世界內的最高規則壓制主角。

可用形式：
1. 權限封鎖
2. 身份抹除
3. 資源凍結
4. 火力鎖定
5. 系統判死
6. 群體審判
7. 官方宣判
8. 數據降級
9. 生存空間壓縮

正文要求：
不要直接寫「規則壓迫」。
要寫出壓迫如何發生。

範例方向：
- 螢幕上的紅點一個接一個變灰。
- 林燼的身份編號被系統刪除。
- 所有通道同時封死。
- 無人機群轉向。
- 底層居民沉默退後。

---

## Step 2：反派誤判

反派必須相信主角仍被困在同一套規則裡。

反派誤判類型：
1. 以為主角只能逃
2. 以為主角只能求饒
3. 以為主角底牌只是低階道具
4. 以為自己掌握最高權限
5. 以為底層民眾沒有反抗意志
6. 以為主角的沉默代表絕望

正文要求：
反派要具體行動，不要只講狠話。

可用動作：
1. 慢慢喝咖啡
2. 不看主角畫面
3. 隨手按下清除鍵
4. 把主角標記成實驗垃圾
5. 對下屬說不用回報
6. 提前打開勝利報告

---

## Step 3：降維介入

主角力量來自敵方規則之外。

降維感來源：
1. 敵方系統無法辨識
2. 權限判定失敗
3. 防禦邏輯自我矛盾
4. 最高指令被更高層指令覆蓋
5. 敵方資料庫出現空白
6. 世界底層出現未知語言
7. 百萬信徒意志匯入主角體內
8. 古老文明主機接管現實規則

正文要求：
先寫異常，再補少量原因。
不要長篇解釋系統設定。

錯誤寫法：
文明主機的原理是透過信仰網路與因果律重新編譯世界底層代碼。

正確寫法：
顧衡的螢幕忽然黑了一瞬。
下一秒，所有紅色鎖定框同時扭曲。
它們沒有消失。
它們像是看見了什麼不該存在的東西，開始瘋狂顫抖。

---

## Step 4：絕對碾壓

外掛開啟後不寫苦戰。

禁止：
1. 來回拆招
2. 長時間僵持
3. 主角重傷硬撐
4. 敵人臨時升級拖戲
5. 用數值計算拖慢節奏

必須：
1. 主角從容
2. 敵方失控
3. 防禦瞬間崩潰
4. 結果乾脆
5. 畫面有衝擊力

可用畫面：
1. 無人機像斷線木偶墜落
2. 防禦牆碎成白色光屑
3. 警報聲突然中斷
4. 鎖定框一格格熄滅
5. 敵方控制台全部回傳失敗
6. 林燼從爆炸中心走出，衣角沒有沾灰

---

## Step 5：震撼反饋 + 實際收穫

爽點需要反饋，也需要收穫。

反饋來源：
1. 反派崩潰
2. 旁觀者失聲
3. 盟友重新認識主角
4. 底層民眾開始信仰主角
5. 更高層敵人第一次注意主角

收穫類型：
1. 新權限
2. 新資源
3. 新信徒
4. 敵方漏洞
5. 新情報
6. 反派失勢
7. 底層聲望提升
8. 更高層主線開啟

反派台詞方向：
1. 這超出了權限。
2. 系統沒有這個等級。
3. 他不是底層編號。
4. 這不是母艦的語言。
5. 停下，立刻切斷連線。
6. 為什麼清除指令在服從他？
