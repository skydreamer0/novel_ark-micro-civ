# 章節品質閘門

> 每寫完一章，在視為完成稿之前，必須通過以下三道閘門。

---

## 第一閘：視角鎖定

- [ ] 這章是誰的視角？一句話答得出來
- [ ] 全章 90% 以上段落沒有跳出這個角色的感知
- [ ] 如果有視角切換，有 `---` 分隔線，且新視角在 2 行內被識別
- [ ] 沒有以下句型（上帝手指）：
  - 「他不知道，此時的……」
  - 「這不是……，是……」（作者代替讀者總結）
  - 替全隊下狀態總結（「這支隊伍像一架破機」）

## 第二閘：資訊不對稱

- [ ] 章中至少有一個資訊差被利用（A 知道 X 而 B 不知道，或相反）
- [ ] 對話中沒有人解釋對方已知的事
- [ ] 至少一句台詞暴露了說話者的認知邊界（而不是純粹傳遞資訊）
- [ ] 角色保留了一句不說的判斷（張力來源）

## 第三閘：感知寫作

- [ ] 沒有任何情緒被直接命名（恐懼/憤怒/震驚/寒意）
  - 替換成：身體反應、動作、環境變化
- [ ] 每一段至少包含：環境 + 身體 或 環境 + 時間感 的兩層覆蓋
- [ ] 系統提示不像遊戲面板（禁止：經驗值/等級/冷卻時間）
- [ ] 系統提示像高維文明介入（如【權限判定中】【檢測到未授權進程】）

## 第四閘：場景化 vs 摘要

- [ ] 每一段寫的是「當下發生的事」，不是回顧/總結/路線板
- [ ] 沒有「很快」「幾分鐘後」「一週內」等時間壓縮詞（必須展開或刪除）
- [ ] 沒有以下禁忌：
  - 行動路線板（「31-40：保鏈、生存、反風控」）
  - 章節總結句（「這個階段的收尾不是大爆炸」）
  - 企劃語（「本章目標」「三線推進」「戰術爽點」）
  - 後設用語（「本章」「下一章」「讀者」「作者」）

## 第五閘：結尾鉤子

- [ ] 最後一段不是結論
- [ ] 結尾符合以下五種之一：
  - 未回答的問題
  - 錯誤的確定（角色下了一個讀者知道資訊不足的判斷）
  - 身體反常（角色做了無法立即解釋的動作）
  - 外部變化（新威脅訊號出現）
  - 沉默的間隔（對話留下的空白比對話本身重）
- [ ] 沒有以宣示/口號/決心做結尾（「那我們就讓全城失步」→ 這是結論，不是鉤子）

---

## 第六閘：字數與結構

- [ ] 正文 ≥ 3000 字（不含章名、空白行、作者註記）
- [ ] 常態範圍 3200～4500 字（若超過需確認非水字數）
- [ ] 反派在文中有出現具體的反饋（至少一個畫面）
- [ ] 爽點：每 800 字至少一次情緒波動

---

## 不合格處理

1. 任一閘門的任一項目為「否」，該章不得視為完成稿
2. 退回修改，修改後重新跑全部閘門
3. 連續兩次不合格 → 先讀 `01_RULES/06_style_dna.md` 和 `01_RULES/05_視角與資訊法.md` 再改寫
