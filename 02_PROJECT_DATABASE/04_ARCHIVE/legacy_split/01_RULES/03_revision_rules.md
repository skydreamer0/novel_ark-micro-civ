# 修訂與審稿規則

> 本文件定義正文完稿後的修訂流程、審稿標準與品質門檻。

---

## 1. 修訂流程

每章完稿後，依序執行：

1. **自查**：對照 `01_NOVEL_WRITING_RULES.md` 的核心目標，確認輸出像小說正文。
2. **修訂**：使用 `00_GLOBAL_RULES.md` 的修訂 Prompt 進行強化。
3. **審稿**：使用 `00_GLOBAL_RULES.md` 的審稿 Prompt 進行評分。
4. **狀態更新**：使用 `00_GLOBAL_RULES.md` 的狀態更新 Prompt，更新 `07_CHAPTER_STATES/CURRENT_STATE.md`。

---

## 2. 修訂必查清單

- [ ] 是否刪除了所有像「遊戲結算」的清單？
- [ ] 系統提示是否像高維文明介入，而非遊戲面板？
- [ ] 是否至少有一個段落讓讀者感受到環境的氣味或觸感？
- [ ] 配角的行動是否有其獨立動機？
- [ ] 反派是否先傲慢、再錯愕、再恐懼？
- [ ] 主角開外掛後是否乾脆碾壓，沒有苦戰？
- [ ] 結尾是否留下鉤子？

---

## 3. AI 味檢測項目

以下任一項出現，即判定為「有 AI 味」，必須修訂：

1. 連續使用三個以上四字成語。
2. 出現「總而言之」「值得注意的是」「不可否認」等論述語氣。
3. 角色台詞風格雷同，分不清誰在說話。
4. 出現括號標註（如：[規則壓迫]、[降維介入]）。
5. 段落結尾用總結性語句收束（如「這場戰鬥讓所有人都明白了……」）。
6. 用形容詞堆砌代替具體畫面（如「極其震撼的場景」）。
7. 連續兩段以上沒有人物動作或環境變化。

---

## 4. 品質門檻

一章正文必須同時通過以下門檻才可定稿：

| 項目 | 門檻 |
| --- | --- |
| 大綱感 | 0（完全沒有） |
| 任務報告感 | 0（完全沒有） |
| AI 味 | ≤ 1 處 |
| 爽點密度 | 每 800 字至少 1 次情緒波動 |
| 反派反饋 | 至少 1 次具體的崩潰或誤判畫面 |
| 章末鉤子 | 必須存在 |
