## 第一卷：艙底種火（1～42 章）

### 核心命題
底層維修工林燼在 72 小時欠稅死亡倒數中，意外啟動體內的「文明主機」，從被收割的螻蟻，變成被高層標記為「未授權文明載體」的危險變數。

### 重大轉折

1. **系統清退倒數**（第1章）
   林燼收到生存稅催繳，系統判定其為「低價值進程」，啟動 72 小時清退序列。

2. **裂縫數據採集與進程衝突**（第2-5章）
   接黑市任務採集原始數據，遭隊長賀鐵背刺，肉體硬體損毀度達 90%。

3. **寄生系統（文明主機）開機**（第5-6章）
   主機開機，開始對肉體硬體進行底層修復，種火內核版本 1.0 上線。

4. **數據截流與偽裝套利**（第7-10章）
   首次截流成功，利用權限漏洞洗白收益，反哺模組覆寫局部物理規則完成反殺。

5. **多進程同步與神殿雛形**（第11-17章）
   燕九、蘇嵐加入，形成協同進程網路，主機升級至神殿時代 Beta 版。

6. **人口進程起源真相**（第20章）
   取得投放碼片段，證實底層人口為系統生成的「批量進程」，無獨立權限。

7. **權限收編與數據清洗**（第21-23章）
   系統嘗試收編（鎖死權限），主角拒絕，舊據點遭遇「物理格式化（清洗）」。

8. **深層遺跡與管理指令（規則碎片）**（第24-26章）
   取得規則碎片，完成神殿時代內核升級，具備改寫局部 API 能力。

9. **夜襲與邏輯炸彈引爆**（第27-29章）
   夜襲節點刪除自身標籤，引發系統邏輯衝突，艙段被迫重啟 6 小時。

10. **未授權載體標籤**（第30章）
    卷末揭示：主角被系統定義為「未授權文明載體（超級病毒）」。

11. **灰鏈網路與能級對抗**（第31-41章）
    建立「灰鏈」非授權網路，對抗系統差異化稅率。

12. **十七分鐘權限爭奪**（第42章）
    利用 R-17 令牌爭奪核心數據。灰鏈被定義為「制度級異常進程」。

## 第二卷：裂縫稅戰（43～60 章）

### 核心命題
從個人求生升級為制度對抗。主角要在「被標記後仍可盈利」的前提下，奪取裂縫入口控制權，最終撕開抽成同盟的遮羞布。

### 重大轉折

1. **標記後的城市稅戰**（第43-50章）
   舊通道失效，測試灰色直煉，形成「低暴露盈利模型」。

2. **裂縫控制權爭奪**（第51-55章）
   從被動納稅轉向爭奪裂縫入口節點，神殿軍規落地。

3. **收編劫車與證據公開**（第56-58章）
   劫回被收編盟友，取得抽成同盟完整證據並上鏈公開。

4. **單艙段稅流停擺**（第59-60章）
   主角引發 12 小時稅流停擺，高層觀察員簽發「二級收割預案」。
