# 章節節奏模板

> 模板用途：設計 3 章一循環的具體節奏——每章的功能、段落分配、視角選擇與開場技巧。
> 本模板是「微循環節奏規則」（01_RULES 第5條）的可操作版本。

---

## 1. 三章循環結構

### 循環定義

| 位置 | 功能 | 建議字數 | 情緒曲線 | 鉤子強度 |
|---|---|---|---|---|
| 第1章 行動章 | 主角發起行動，建立新壓力 | 3200~3800 | 中→高→中 | 3~4 |
| 第2章 衝突章 | 阻力爆發，雙方碰撞 | 3800~4500 | 高→更高→更高 | 4~5 |
| 第3章 結算章 | 清理戰場，盤點收益，設定新目標 | 3200~3800 | 高→中→懸念 | 3 |

### 循環節奏曲線（情緒溫度示意）

```
行動章：    ▃▅▇▅▃    （啟動→升溫→降溫預備）
衝突章：    ▅▇█▇▇    （直接高壓→持續壓迫→更高壓收尾）
結算章：    ▇▅▃▅▃    （降溫→盤點→新懸念升起）
```

## 2. 行動章設計（循環第1章）

### 功能
主角主動出擊，設定目標，展示策略。

### 內部段落分配（以 3500 字為例）

| 段落區段 | 建議字數 | 內容 |
|---|---|---|
| 開場接鉤子 | 0~500 | 承接上一章結尾，500字內回應上一章留下的懸念 |
| 目標確認 | 500~1000 | 主角確認當前目標，團隊分工，調度資源 |
| 執行行動 | 1000~2500 | 行動展開——潛入、談判、掠奪、滲透 |
| 意外發生 | 2500~3000 | 行動中出現未預期的阻力或新情報 |
| 結尾鉤子 | 3000~3500 | 行動的結果導致一個新的、更大的問題浮現 |

### 開場技巧

**不要做的事**：從角色起床開始、從角色回憶開始、從天氣描述開始。

**要做的事**：直接從上一章的鉤子切入。

> 上一章結尾：蘇嵐看見林燼腕環上的審計追蹤碼。
> 下一章開場：
> 「你被標記了。」
> 蘇嵐的聲音從通訊頻道傳進來，沒有疑問句的尾音。
> 林燼低頭看了一眼腕環。他知道。他只是還沒想好怎麼處理。

### 視角選擇

行動章建議視角優先級：
1. **林燼**（最常用——行動章需要主角的決策視角）
2. **蘇嵐**（當行動需要技術判斷，且林燼不在場）
3. **燕九**（行動是戰鬥為主，且林燼在另一條線）

## 3. 衝突章設計（循環第2章）

### 功能
阻力正式爆發，雙方直接碰撞，緊張度到達循環頂峰。

### 內部段落分配（以 4000 字為例）

| 段落區段 | 建議字數 | 內容 |
|---|---|---|
| 開場接鉤子 | 0~500 | 上一章結尾的壓力在此處引爆 |
| 衝突升級 | 500~1500 | 反派或環境阻力加碼，主角被壓制 |
| 中場反轉 | 1500~2500 | 主角找到突破口，或新變數介入扭轉局勢 |
| 持續壓迫 | 2500~3500 | 即使反轉，代價或副作用正在累積 |
| 結尾鉤子 | 3500~4000 | 衝突的結論不是結束——更高層次的威脅已經啟動 |

### 中場反轉的設計

衝突章的第 1500~2500 字區間必須有一個反轉。反轉類型：

- **系統反轉**：主機完成某項運算，改變局面
- **資訊反轉**：蘇嵐破解敵方通訊，暴露敵方弱點
- **環境反轉**：戰鬥中的物理環境被利用（管道爆炸、壓力門關閉）
- **第三方介入**：旁觀勢力突然參戰（不是機械降神，而是之前鋪墊過的勢力）

> 林燼被壓在火力封鎖線後方。
> 對方的彈藥像是無限的——他計算過，以這個射速，對方再撐九十秒就會耗盡。
> 但九十秒後，他的掩體也會被打穿。
> 然後他的腕環震了。
> 不是通知——是蘇嵐。她只說了三個字：
> 「往右走。」
> 他沒有問為什麼。他往右滾了兩個身位。
> 然後他剛才蹲著的位置——天花板塌了。不是巧合。蘇嵐在他往右滾的前一秒，炸掉了上層的結構節點。

### 視角選擇

衝突章建議視角優先級：
1. **林燼**（正面衝突時——壓迫感最強）
2. **蘇嵐**（衝突是資訊戰或技術戰時）
3. **第三方視角**（罕用——只在需要展示反派反應時使用，如顧衡視角的衝突章）

## 4. 結算章設計（循環第3章）

### 功能
衝突結束後的盤點——清理戰場、結算收益與損失、設定下一循環的目標。

### 內部段落分配（以 3500 字為例）

| 段落區段 | 建議字數 | 內容 |
|---|---|---|
| 開場接鉤子 | 0~500 | 確認上一章衝突的結果——誰活了，誰死了，什麼沒了 |
| 財報盤點 | 500~1500 | 投入→轉化→產出的具體清單（場景化，非列表） |
| 團隊反應 | 1500~2500 | 角色之間因收益分配或代價承擔產生對話張力 |
| 主機/內部文明 | 2500~3000 | 主機層面的反饋——煉化進度、鏈路狀態、內部動向 |
| 結尾鉤子 | 3000~3500 | 盤點結束後，下一階段的目標浮現或更高威脅逼近 |

### 財報盤點的寫法

**禁止**：
> 投入：120份樣本，40單位信仰
> 轉化：主機煉化折損率12%
> 產出：3台戰車，2套裝備

**正確（場景化）**：
> 林燼站在倉庫中央。
> 地上排著三台剛組裝完成的戰車——外殼還帶著焊槍的餘溫，其中一台的履帶護板裝反了，燕九正在拆掉重裝。
> 「三台能用。」蘇嵐的聲音從戰車底盤下面傳出來。「材料耗掉七成——以這個轉化率，下一次要湊到同樣的量，至少需要五天。」
> 林燼沒有回答。
> 他在算：五天的空窗期，自治窗口還剩多久。

### 視角選擇

結算章建議視角優先級：
1. **林燼**（需要他的決策視角來鋪陳下一循環）
2. **蘇嵐**（當結算重心在技術資源盤點時）
3. **唐雀**（結算重心在資訊或數據分析時）

### 結算章的常見錯誤

- ❌ 寫成流水帳（先後做了A、B、C）
- ❌ 沒有新目標（盤點完就結束，沒有指向未來）
- ❌ 角色之間沒有衝突（所有人都同意結算結果——太假）
- ✅ 正確：結算中必須有分歧——關於下一步怎麼走、資源怎麼分、誰來承擔代價

## 5. 視角選擇指南

### 三條視角選擇軸線

**軸線A：距離衝突遠近**
- 近距離衝突（戰鬥、談判、潛入）→ 林燼
- 中距離支援（技術破解、監控、黑市交易）→ 蘇嵐或唐雀
- 遠距離觀測（敵方動向、全局判斷）→ 任何角色皆可，但必須鎖定一個

**軸線B：資訊量需求**
- 需要讀者知道主角的內部判斷 → 林燼
- 需要保留主角的內部判斷（製造懸念） → 蘇嵐或燕九
- 需要展示反派視角 → 顧衡（每 5~8 章一次，不建議更頻繁）

**軸線C：這一章的主題**
- 決策/策略 → 林燼
- 技術/資訊 → 蘇嵐
- 戰鬥/行動 → 燕九
- 系統/制度 → 唐雀

### 第121章續寫視角建議

基於當前進度（第120章狀態）：
- 建議第121章以**蘇嵐視角**開場——她在灰槽帶外圍監控點，第一個看到白名單公告
- 第122章切回**林燼視角**——他需要決定如何應對白名單制
- 第123章結算——**林燼視角**盤點灰槽帶資源與暴露風險

## 6. 開場技巧（不要從角色起床開始）

### 四種有效開場

1. **從對話中間切入**
> 「你不應該在這裡。」
> 蘇嵐的聲音從通訊頻道傳來，比平時低了半個音階——她在壓著情緒說話。

2. **從身體狀態切入**
> 林燼的左腿在痛。
> 不是受傷的那種痛——是義肢接口的磨損警報。他已經連續走了十九個小時。

3. **從環境異常切入**
> 灰槽帶的燈，在凌晨三點同時滅了。
> 不是斷電——備用照明還亮著。
> 是有人關了主照明。

4. **從正在進行的動作切入**
> 林燼把最後一枚螺絲鎖緊。
> 戰車的引擎蓋啪一聲闔上。他沒有試發動——他不需要試，他知道它能動。

### 開場自檢
- [ ] 前 200 字是否有一個具體的感官細節？（光線、聲音、氣味、觸感、痛覺）
- [ ] 前 500 字是否出現了當前的問題或目標？
- [ ] 開場是否直接承接了上一章的鉤子？（如果不是，為什麼？）

## 7. 結尾鉤子強度遞增

### 循環內鉤子強度分配

| 循環位置 | 鉤子強度 | 功能 |
|---|---|---|
| 行動章結尾 | 3~4 | 讓讀者想看下一章的衝突 |
| 衝突章結尾 | 4~5 | 讓讀者等不及結算章的盤點 |
| 結算章結尾 | 3 | 留懸念，但讓讀者有喘息空間 |

### 跨循環鉤子遞增

每完成一個完整的 3 章循環，下一循環的基礎張力應該比前一循環更高。具體實現方式：

- 循環1結束時：解決了一個戰術問題，但發現了更大的戰略問題
- 循環2結束時：戰略問題升級為生存問題
- 循環3結束時：生存問題升級為不可逆的制度變革
