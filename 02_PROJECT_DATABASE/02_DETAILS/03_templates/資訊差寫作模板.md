# 資訊差寫作模板

> 模板用途：利用角色間的資訊不對稱製造張力——每場對話、每次衝突之前，先釐清誰知道什麼、誰不知道什麼。
> 本模板與 `01_RULES/05_視角與資訊法.md` 互補：該檔定義原則，本檔提供可操作的表格、範例修改與當前設定。

---

## 1. 資訊差的三個層級

### 戰術級（即時——影響接下來幾分鐘到幾小時）

| 層級 | 範圍 | 影響 |
|---|---|---|
| 誰在哪裡 | 角色位置與動向 | 伏擊、包圍、救援的成敗 |
| 誰有什麼 | 當前裝備、資源、狀態 | 戰鬥力的即時判斷 |
| 權限狀態 | 當前可用權限與鎖定 | 通行、操控、關閉系統的能力 |

### 戰略級（中期——影響接下來幾章）

| 層級 | 範圍 | 影響 |
|---|---|---|
| 敵方意圖 | 反派真正的目標 | 主角判斷失誤的代價 |
| 規則漏洞 | 制度系統的弱點 | 能否在制度內找到突破口 |
| 資源鏈 | 補給來源與分配模式 | 長期生存能力的判斷 |
| 暴露風險 | 誰被追蹤、誰即將暴露 | 安全與否的根本判斷 |

### 情感級（長期——影響角色關係與伏筆回收）

| 層級 | 範圍 | 影響 |
|---|---|---|
| 隱瞞的事實 | 角色主動隱瞞的資訊 | 信任裂痕的來源 |
| 創傷與動機 | 角色行為的深層原因 | 角色關係變化的基礎 |
| 代價真相 | 某個行動的真正代價 | 事後反噬的伏筆 |

## 2. 角色資訊不對稱表（基於第120章狀態）

### 當前核心資訊差矩陣

| 資訊項 | 林燼知道 | 蘇岚知道 | 燕九知道 | 顧衡知道 |
|---|---|---|---|---|
| V9 暴露概率（5%~70%） | ✔ | ✘ | ✘ | ✘（但懷疑） |
| 灰槽帶淨化符庫存為零 | ✔ | ✔ | ✔ | ✔ |
| 民生探針接收方身份 | ✘（正在查） | ✘ | ✘ | ✔ |
| 探針已被監控（明哨+暗樁） | ✔ | ✔ | ✘ | ✔ |
| 白名單生存制具體內容 | ✘（公告未出） | ✘ | ✘ | ✔（起草者） |
| 祭司派與收割派外圍聯繫 | ✔（懷疑中） | ✘ | ✘ | ✘ |
| 顧衡啟動第七層協議 | ✘ | ✔（聽到錄音） | ✘ | ✔ |
| 主機鏈路衰減程度 | ✔ | ✘ | ✘ | ✘ |
| 自治權撤銷條款觸發條件 | ✔ | ✔ | ✘（沒細讀） | ✔ |
| 白扣被俘後下落 | ✘（伏筆） | ✘ | ✘ | ✔ |

### 當前最大資訊差帶（第121章可用）

1. **林燼 vs 蘇岚**：林燼知道 V9 暴露概率已近警戒線，但沒告訴蘇岚。蘇岚的後續判斷建立在「林燼還能再用一次主機」的錯誤前提上。
2. **林燼 vs 顧衡**：顧衡知道白名單制的全部細節，但不知道林燼的主機可以操縱名冊資料。林燼知道顧衡啟動了第七層協議，但不知道第七層的具體內容。
3. **蘇嵐、燕九 vs 林燼**：兩人都不知道祭司派與收割派外圍有關聯。林燼選擇暫時不告訴他們——因為他還沒確認。

## 3. 對話中保留資訊的技巧

### 核心原則：角色不該解釋對方已知的事

### 技巧A：省略已知資訊

| ❌ 錯誤（蘇岚對林燼解釋） | ✅ 正確（蘇岚只說增量） |
|---|---|
| 「這是審計中樞的備用接口路由，不是常規格式——代表顧衡透過第三方的通道在送資料。」 | 「路由不對。」蘇岚把面板推過來。她沒說哪裡不對——因為她還在確認自己的判斷。 |

### 技巧B：用問題代替告知

當角色需要確認對方知道某件事時，不要用陳述句告知，用疑問句試探。

> 「你看到公告了？」
> —— 不是「白名單生存制已經發布了」。

### 技巧C：保留判斷，只給事實

角色不一定要說出他們對某件事的判斷。

> 「探針的信號經過三層跳轉，消失在民用網和中層政務網交界處。」
> —— 蘇岚沒有說「這代表接收方具有中層以上權限」，因為林燼自己會推出來。

### 技巧D：保留不說本身就是張力

角色知道的事情，不一定要在當下說出來。

> 林燼沒有告訴蘇岚，他的 V9 暴露概率已經到了 35%。
> 不是因為他忘了。
> 是因為說出來之後，蘇岚的決策會偏向保守——而他需要她在接下來的行動中保持激進。

## 4. 感知寫作法的具體應用

### 從「情緒命名」到「身體反應」

| 情境 | ❌ 情緒命名 | ✅ 感知寫作 |
|---|---|---|
| 蘇岚警覺 | 蘇岚眼底全是寒意 | 林燼看見她的食指在扳機護圈外側敲了一下 |
| 林燼不滿 | 他心裡冷笑 | 他沒回話 |
| 顧衡震驚 | 顧衡很震驚 | 顧衡的手停在半空。咖啡杯裡的黑色液面晃了一圈。 |
| 蘇岚緊張 | 蘇岚很緊張 | 她說這話時，那截銀色義指在槍管上敲了兩下 |
| 燕九壓怒 | 燕九壓不住怒火 | 燕九把彈匣拍進握把，那聲響比平時重了一倍 |

### 感知三層強制練習

寫作任何一段前，先填以下三欄（至少填兩欄）：

```
環境感官：（溫度/氣味/光線/聲音）
身體反應：（痛/麻/延遲/渴/疲憊）
時間壓力：（倒數/等待/呼吸次數）
```

**範例**：
> 環境：管道間的空氣帶著焊渣味和舊線路加熱後的焦臭
> 身體：林燼的膝蓋義肢在蹲下時發出輕微的摩擦聲——接口磨損了
> 時間：距離下一次權限同步還有 14 分鐘

## 5. 範例修改：普通寫法 vs 資訊差寫法

### 場景：蘇岚發現林燼被追蹤

**普通寫法（資訊差未利用）：**
> 蘇岚看著林燼腕環上的追蹤碼，心裡一沉。她知道那是審計中樞的內部追蹤碼——林燼被標記了。但她不知道林燼已經知道了這件事。
> 「你被標記了。」她說。
> 「我知道。」林燼回答。
> 「你為什麼沒告訴我？」
> 「我還沒想好怎麼處理。」
> 蘇岚沉默了。她覺得自己被排除在決策之外，這讓她很不舒服。

**資訊差寫法（保留資訊+感知寫作）：**
> 蘇岚看見了那串編碼。
> 她的視線在林燼的腕環上停了兩秒——比她平時確認一組數據需要的時間，多了整整一秒半。
> 她沒有問那是什麼。
> 她認得那串編碼的開頭字段。
> 她把工具箱闔上。發出比平時更重的一聲。
> 那聲響在狹窄的管道間迴盪了很久。
> 林燼沒有解釋。
> 她也沒有再問。

### 差異分析

| 面向 | 普通寫法 | 資訊差寫法 |
|---|---|---|
| 資訊揭露 | 蘇岚直接說出判斷 | 蘇岚不說判斷——她的沉默就是判斷 |
| 情緒表現 | 直接命名（心裡一沉、不舒服） | 身體反應（停兩秒、闔工具箱更重） |
| 對話量 | 4 句對話，過度交流 | 0 句有效對話——交流在沉默中完成 |
| 讀者參與度 | 被動接收資訊 | 主動推測蘇岚的情緒和林燼的沉默 |
| 角色個性 | 蘇岚在質問，不像蘇岚 | 蘇岚用行動表達不滿——像蘇岚 |

### 第二組範例：顧衡發現權限被鎖

**普通寫法：**
> 顧衡驚訝地發現自己的清除權限被鎖住了。他不敢相信這是真的。他又試了三次，但系統每次都回傳同樣的錯誤。他感到非常恐懼，因為這代表對方比他的權限更高。

**資訊差寫法：**
> 【錯誤：執行者權限低於被清除者】
> 顧衡看著那行紅字。
> 他沒有驚慌——他的第一個反應是指令輸錯了。
> 他重新輸入了一遍。同樣的結果。
> 他的手指在鍵盤上停了一瞬——那一瞬比他預期的長了一半。
> 他沒有輸入第三次。
> 因為他已經知道，第三次不會有不同的結果。

## 6. 資訊差實戰設計表

### 每場衝突前的 3 分鐘準備

**步驟1：列出在場角色**
角色 A：______
角色 B：______
角色 C：______

**步驟2：畫出知道/不知道矩陣**

| 關鍵資訊 | A 知道 | B 知道 | C 知道 |
|---|---|---|---|
| 例：V9 暴露風險 | ✔ | ✘ | ✘ |
| 1. ______ | | | |
| 2. ______ | | | |
| 3. ______ | | | |

**步驟3：定義至少一條「不說」**

角色 _____ 知道 _____，但選擇不告訴 _____，因為 _____。

**步驟4：定義對話中暴露資訊邊界的時刻**

當 _____ 說 _____ 的時候，_____ 意識到對方不知道 _____。

## 7. 資訊差自檢清單

- [ ] 這一章的視角鎖定了誰？（如果你能答出，視角是對的）
- [ ] 哪個角色知道得比其他人多？（如果沒有，資訊差不夠）
- [ ] 哪個角色故意隱瞞了資訊？（如果沒有，少了張力來源）
- [ ] 對話中是否有人解釋了對方已知的事？（如果有，刪掉）
- [ ] 情緒是否被直接命名？（如果有，改成身體反應）
- [ ] 是否至少有 3 個感官細節？（環境、身體、時間感——至少兩個）
- [ ] 是否有一個時刻，讀者比角色更早知道某件事？（這是製造緊張感的利器）
