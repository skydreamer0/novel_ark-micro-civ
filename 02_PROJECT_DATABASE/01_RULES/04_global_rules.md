# 全域規則：AI 寫小說總工作流

> 本文件定義 AI 輔助寫作的工作流程、品質門檻與 Prompt 模板。
> 所有正文生成、修訂、審稿均以本文件為準。

---

## 1. 工作流順序

1. **世界觀檔**
   建立世界規則、階級制度、科技邏輯、禁忌、權限系統。

2. **角色卡**
   建立主角、反派、盟友、旁觀者的人設、目標、恐懼、口頭禪、行動邏輯。

3. **主線大綱**
   只記錄重大轉折，不寫正文。

4. **章節細綱**
   每章只列：
   - 本章目標
   - 本章衝突
   - 本章爽點
   - 本章結尾鉤子

5. **正文生成**
   每次只寫一章或一個場景。
   不一次生成太多章，避免崩設定。

6. **修訂**
   修訂節奏、人物反應、爽感密度、場景畫面。

7. **狀態更新**
   每章結束後更新：
   - 角色狀態
   - 新增設定
   - 已揭露伏筆
   - 未解謎團
   - 敵方反應
   - 主角新能力或新資源

---

## 2. AI 感消除規則

AI 常見問題：
1. 太愛總結
2. 太愛解釋
3. 太愛用形容詞
4. 太愛用四字詞
5. 太像企劃書
6. 爽點只寫概念
7. 角色台詞都像同一個人
8. 章節結尾沒有鉤子

修正要求：
1. 少用抽象詞，多寫具體畫面。
2. 少寫設定，多寫事件。
3. 少寫情緒名稱，多寫身體反應。
4. 少寫「他感到恐懼」，改寫「他的喉結滾了一下」。
5. 少寫「系統很強」，改寫「最高權限被反向鎖死」。
6. 少寫「眾人震驚」，改寫「原本跪在地上的人一個個抬起頭」。

## 3. 章節生成 Prompt

你是中文網路小說正文寫手。

請根據以下資料生成一段中文網路小說正文。

### 寫作目標
把大綱改寫成真正的小說正文。
不要寫大綱。
不要寫分析。
不要寫括號標註。
不要解釋爽點。
不要使用企劃書語氣。

### 類型
科幻末世、階級壓迫、系統外掛、降維打臉、無敵爽文。

### 本段爽感架構
1. 規則壓迫
2. 反派誤判
3. 降維介入
4. 絕對碾壓
5. 震撼反饋 + 實際收穫

### 正文要求
1. 用場景寫作。
2. 用動作呈現情緒。
3. 反派要先傲慢，再錯愕，再恐懼。
4. 主角少說話。
5. 系統異常先出現，原因後揭露。
6. 開外掛後不苦戰。
7. 每 300 至 500 字要有一次情緒轉折。
8. 結尾要留下更高層威脅或新伏筆。

### 禁止
1. 禁止寫「規則壓迫」「降維介入」「碾壓」「震撼反饋」這些標籤。
2. 禁止寫成條列。
3. 禁止像任務日報。
4. 禁止過度解釋設定。
5. 禁止讓主角開外掛後陷入苦戰。

## 4. 修訂 Prompt

請修訂以下小說正文。

### 修訂目標
讓文字更像中文網路小說正文，強化爽感、畫面感、壓迫感、反派反饋。

### 必須修正
1. 刪除大綱感。
2. 刪除說明感。
3. 刪除企劃書語氣。
4. 增加具體畫面。
5. 增加人物動作。
6. 增加反派誤判。
7. 增加系統異常。
8. 增加旁觀者或敵方反饋。
9. 主角開外掛後要乾脆碾壓。
10. 結尾補一個鉤子。

### 禁止
1. 不要新增長篇設定解釋。
2. 不要讓主角苦戰。
3. 不要寫成條列。
4. 不要保留括號標籤。
5. 不要用評論語氣。

### 輸出格式
直接輸出修訂後正文。

## 5. 審稿 Prompt

請檢查以下章節是否符合中文網路爽文要求。

### 檢查項目

1. 是否像小說正文
2. 是否有大綱感
3. 是否有任務報告感
4. 反派是否足夠傲慢
5. 壓迫是否足夠具體
6. 主角出手是否夠乾脆
7. 系統外掛是否有降維感
8. 反派反應是否足夠震撼
9. 是否有旁觀者反饋
10. 打臉後是否有實際收穫
11. 結尾是否有鉤子
12. 是否有 AI 味

## 6. 狀態更新 Prompt

請根據本章內容更新小說狀態檔。

### 需要更新

1. 本章發生了什麼
2. 主角目前狀態
3. 反派目前狀態
4. 新增能力
5. 新增設定
6. 已揭露伏筆
7. 未解伏筆
8. 主角獲得的資源
9. 敵方損失
10. 下一章可接的衝突

### 輸出格式

```
# CHAPTER_STATE_UPDATE

## 本章摘要

## 角色狀態

## 世界觀新增

## 能力系統新增

## 伏筆

## 下一章銜接
```
