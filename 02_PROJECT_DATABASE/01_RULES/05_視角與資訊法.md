# 角色資訊視角法

> 目標：讓每個角色帶著不同的資訊量進入每一場對話與衝突。
> 核心原則：**讀者只能透過一個角色的感知理解世界，其他角色的內心是盲區。**

---

## 第一層：視角鎖定規則

### 規則 1A：每章選一個「感知主體」

全章 90% 以上的段落，只能寫這個角色能看到、聽到、聞到、感覺到、想到的東西。

| 感知主體 | 讀者能知道 | 讀者不能知道 |
|---|---|---|
| 林燼 | 他觀察到的蘇嵐手指在敲桌面 | 蘇嵐在想什麼 |
| 蘇嵐 | 她看到林燼沉默了三秒 | 林燼為什麼沉默 |
| 顧衡 | 數據異常跳出紅字 | 林燼在廢區哪個角落 |

### 規則 1B：避免「上帝手指」

禁止寫以下句型——它們是全知視角的標誌：

- 「他不知道，此時的顧衡已經……」
- 「這支隊伍像一架仍能飛、但每塊蒙皮都在響的破機」
- 「這不是追捕，是讓城市本身變成一張網」

**改法**：把判斷權還給讀者。只寫角色知道的，不替讀者總結。

### 規則 1C：視角轉移要有斷點

必須換視角時，用 `---` 分隔線明確切換，並在開頭兩行內讓讀者知道現在是誰的視角。

## 第二層：資訊不對稱戰術

### 規則 2A：畫三角圖——誰知道什麼

在任何對話場景前，快速建立一個三角圖：

```
林燼知道：主機鏈路在衰減、H-12還能撐多久
蘇嵐知道：弟弟被企業控制的最新動向
燕九知道：傷口其實比報告中更嚴重（但他沒說）

→ 對話時，三人對同一件事的理解完全不同
```

### 規則 2B：用對話暴露資訊邊界

角色不該在對話中解釋對方已知的事情。

| ❌ 錯誤（資訊流通太順暢） | ✅ 正確（對話暴露認知差） |
|---|---|
| 「顧衡把處置權下放給區域代理了，這代表我們會面對更碎片化的敵人。」 | 「下放了。」林燼說。蘇嵐皺眉——她在等解釋。他沒給。 |
| 「這是審計中樞的備用接口路由，不是常規格式。」 | 「路由不對。」蘇嵐把面板推過來。她沒說哪裡不對——因為她還在確認自己的判斷。 |

### 規則 2C：保留「角色不說的話」

角色知道的事情，不一定要在當下說出來。保留資訊本身就是張力。

林燼發現反應速度異常時沒告訴蘇嵐 → 蘇嵐的後續判斷建立在錯誤資訊上 → 衝突自然產生。

**寫法技巧**：每次角色開口前，先想——他/她為什麼要在這時候說這個？如果沒有非說不可的理由，讓他/她閉嘴。

## 第三層：感知寫作法（代替情緒命名）

### 規則 3A：禁止直接命名情緒

| ❌ 禁止 | ✅ 替代 |
|---|---|
| 蘇嵐眼底全是寒意 | 林燼看見她的食指在扳機護圈外側敲了一下 |
| 他心裡冷笑 | 他沒回話 |
| 林燼眼神一沉 | 他眨眼的頻率慢了一拍 |
| 蘇嵐語氣比金屬還冷 | 她說這話時，那截銀色義指在槍管上敲了兩下 |
| 顧衡很震驚 | 顧衡的手停在半空，咖啡杯裡的黑色液面晃了一圈 |

### 規則 3B：感官三層——環境、身體、時間

每一段正文至少要覆蓋以下三層中的兩層：

1. **環境**：溫度、氣味、光線、聲音（母艦管線的低鳴、空氣裡的焊渣味）
2. **身體**：角色的生理反應（延遲、酸、痛、麻、渴）
3. **時間感**：緊迫或停滯（倒數、等待、呼吸次數）

**範例（第31章改寫版）**：
> 林燼靠著爐體坐下。
> 神經延遲在這時候才真正追上他——剛才翻下哨塔時，左腿落地的角度不對。不是痛，是他下令「彎膝」到大腿回應之間，多了一個幾乎覺察不到的空白。
> 那個空白，如果在戰鬥中出現，就是中彈的間距。

## 第四層：場景化寫作（代替摘要）

### 規則 4A：每一段都必須是「當下」發生的事

| ❌ 摘要模式 | ✅ 場景模式 |
|---|---|
| 夜裡九點，三人摸到 H-12 廢區。戰鬥只打了三分鐘。蘇嵐先點掉高處哨兵，燕九切斷電控。 | 林燼蹲在倒塌混凝土牆後方。探照燈的光掃過來——他已經不在那裡。——一個動作接一個動作，讀者跟著時間走。 |
| 回程途中，燕禾收到一條匿名訊息。 | 腕環震了。林燼低頭。螢幕上字很少。 |

### 規則 4B：避免「時間壓縮連接詞」

刪除以下詞彙，把它們展開成場景：

- 「很快」 → 用具體時間或事件間隔取代
- 「幾分鐘後」 → 如果非得跳時間，用角色狀態改變暗示間隔
- 「一週內」 → 選擇其中一場關鍵場景，捨棄其他

### 規則 4C：不能寫進正文的內容清單

- 行動路線板（如「31-40：保鏈、生存、反風控」）
- 章節總結語（如「這個階段的收尾不是大爆炸，而是……」）
- 角色情緒標籤（恐懼、憤怒、震驚——用身體反應代替）
- 大綱式過渡（如「他們很快調整做法，不再……」）

## 第五層：章末鉤子規則

### 規則 5A：鉤子的功能不是「說出結論」

| ❌ 結論式結尾 | ✅ 鉤子式結尾 |
|---|---|
| 「顧衡想要全城風控同步——那我們就讓全城失步」 | 蘇嵐在暗處輕輕哼了一聲。不是嘲諷。是確認。 |
| 「他看著板上字跡，知道路會更難」 | 林燼沒回答。他還沒想好下一步。但讓蘇嵐知道他還沒想好，比讓她知道他在怕還危險。 |

### 規則 5B：五種有效鉤子

1. **未回答的問題**：角色問了一個問題，沒人回答
2. **錯誤的確定**：角色下了一個判斷，讀者知道資訊不足
3. **身體反常**：角色做了一個讀者無法立即解釋的動作
4. **外部變化**：環境出現一個新的威脅訊號
5. **沉默的間隔**：對話結束後留下的空白，比對話本身更重

## 速查：寫完一章後的 5 題自檢

1. 這一章是誰的視角？——如果答不出來，視角亂了
2. 哪個角色知道得比其他人多？——如果沒有，資訊不對稱沒被利用
3. 哪句對話是「角色解釋對方已知的事」？——刪掉它
4. 哪個情緒被直接命名了？——改成身體反應
5. 最後一段是結論還是鉤子？——如果是結論，重寫
