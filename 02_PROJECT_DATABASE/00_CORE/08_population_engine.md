## 一、人口引擎的本質定義

### 阿卡夏熔爐 vs 信仰回流

| 維度 | 信仰回流（V1-V2） | 阿卡夏熔爐（V3+） |
|:---|:---|:---|
| **定義** | 被動接收信徒的信仰能量，轉化為 H-Compute | 主動在虛擬空間中製造人口，轉化為規則改寫權限 |
| **運作方式** | 信徒祈禱 → H-Compute 產出 | 規則碎片 + H-Compute + 投放碼 → 新人口誕生 |
| **人口來源** | 外部現實中的真實人口 | 內部文明中的虛擬人口 + 外部世界的投放批次 |
| **質量控制** | 依賴信徒的虔誠度，易損失 | 可精確設計，品質穩定 |
| **規模上限** | 受現實艙段人口總數限制 | 突破現實物理限制，可無限製造 |
| **代價機制** | 信徒流失、信仰衰減 | 規則碎片消耗、H-Compute 透支、種族熵增 |

### 核心機制：四大製造舱的物理建模

**阿卡夏熔爐**包含四個獨立但互聯的製造空間：

```
[規則碎片分解] → [H-Compute 轉化核] → [種族編碼模板] → [人口輸出緩衝]
     ↓                    ↓                   ↓                 ↓
  人族舱                混合舡                精神體舡           總調度站
  (基礎版)             (增強版)              (虛擬版)          (質量檢驗)
```

每個舱室的獨立成本計算，但共享一個「總 H-Compute 資源池」。

## 二、資源消耗體系完整表

### 規則碎片 → 融合精華的轉化公式

```
融合精華產出量 = (規則碎片品級 × 轉化效率係數) - 種族熵增損耗

其中：
- 規則碎片品級：V1 級（1 單位） ~ V2 級（5 單位） ~ V3 級（20 單位）
- 轉化效率係數：V3 時代的基礎係數 = 0.85（每 10 人製造損失 1.5 單位精華）
- 種族熵增損耗：每製造 1000 人時損失 50~150 單位精華（取決於種族）
```

## 三、四大製造舡完整系統表

### 人族製造舡（基礎版）

| 維度 | 規格 | 說明 |
|:---|:---|:---|
| **原料來源** | 投放碼 × N + 灰槽帶底層人口資訊素 | 從現實世界的人口批次與艙段數據中提取「人口原型」 |
| **生產週期** | 3~5 天/千人 | V3 時代的基礎製造速度 |
| **低端人口特色** | 記憶與個性被削弱至 60%～80% | 快速製造必然伴隨品質下降，製造出來的人口更容易被操縱 |
| **場景細節** | 舡室內部漂浮著蛋形的「孕育囊」，每個囊內部可見模糊的人形輪廓；舡壁上遍佈髮光的文字與圖案（規則碎片的實時轉化過程） | 視覺上像一個「生物工廠」而非「神聖殿堂」 |
| **品質分層** | 等級 D：記憶刪除，基礎服從（成本最低） ➜ 等級 A：完整記憶，自主意識（成本最高） | 林燼可根據戰術需求調整品質 |
| **故障率** | 等級 D：5% 認知崩潰率 ➜ 等級 A：0.2% | 低端人口易在高壓環境中精神分裂 |

### 混血族製造舡（增強版）

| 維度 | 規格 | 說明 |
|:---|:---|:---|
| **原料來源** | 人族原型 + 機械部件編碼 + 神經接口藍圖 | 將人族人口部分機械化，提升耐受力與計算能力 |

### 精神體製造舡（虛擬版）

## 五、與 V9 四項能力的化學反應

### 內世界反饋強化——倍率進化詳表

```
反饋強化倍率的計算公式：

基礎倍率（版本固定）× 信徒虔誠度係數 × 製造舡運作數 × 種族融合度

其中：
- V3 基礎倍率：×2.5
- V4 基礎倍率：×2.8
- V5 基礎倍率：×3.0

信徒虔誠度係數：0.7（低）~ 1.0（中）~ 1.3（高）

製造舡運作數：
  1 個舡 +0.5 倍率
  2 個舡 +0.8 倍率
  3 個舡 +1.1 倍率
  4 個舡 +1.3 倍率

種族融合度：完全分離 ×0.8 ~ 高度融合 ×1.2
```

### 實際應用場景

| 場景 | 計算公式 | 最終倍率 | 說明 |
|:---|:---|:---:|:---|
| V3 單舡運作 | 2.5 × 1.0 × 0.5 × 1.0 | **×1.25** | 基礎場景，信息反饋較弱 |
| V3 四舡全開 | 2.5 × 1.0 × 1.3 × 1.0 | **×3.25** | 林燼可精確預測聖城所有部門的決策 |
| V4 再調和 + 三舡 | 2.8 × 1.1 × 1.1 × 1.1 | **×3.7** | 包含了種族間的協同信號 |
| V5 四族全開 + 高虔誠 | 3.0 × 1.3 × 1.3 × 1.2 | **×6.2** | 理論峰值，但實現難度極高 |

**含義**：倍率越高，林燼對內部文明的「預知能力」越強。在 V5 峰值，他幾乎可以預測未來 24 小時內聖城所有成員的行動。

### 名冊資料操縱的進化路徑

#### V3 時代的應用

```
基礎操縱：
  - 竄改單個人口的身份信息（ID、艙段、工作職位）
  - 偽造投放批次記錄
  - 創造虛擬人口（完全虛構的身份）

高級操縱：
  - 批量注入虛擬投放單位（50~100 人規模）
  - 在系統中創造「人口流動軌跡」，製造假象
  - 操縱投放碼的分配，決定誰進入哪個艙段

範圍限制：
  - 只能操縱第四卷所掌控的三段名冊（第 15、14、12 段）
  - 超出範圍的操縱會立即觸發審計司的異常告警
```

#### V4 時代的應用（預留）

```
強化操縱：
  - 批量注入虛擬製造舡輸出（500~1000 人規模）
  - 在全艦名冊系統中植入「虛擬供應鏈」
  - 操縱整個投放批次的智能分類邏輯

範圍擴張：
  - 可同時操縱 6～8 個艙段的名冊
  - 跨艙段的身份假冒成為可能
```

#### V5 時代的應用（預留至第六卷）

```

```

## 附錄：關鍵公式速查表

### H-Compute 產能基準

```
單個信徒的日均產出 = 基礎產出 × 虔誠度係數 × (1 + 版本加乘)

基礎產出：10~30 單位/天（因個人能力而異）
虔誠度係數：0.5（冷漠） ~ 1.0（中等） ~ 2.0（狂熱）
版本加乘：V1 ×0 | V2 ×1 | V3 ×2.5 | V4 ×2.8 | V5 ×3.0
```

### 規則碎片消耗速率

```
月度碎片消耗 = 月度人口產出 / 製造效率 + 種族熵增補償 + 系統開銷

V3 標準：(600,000 人 / 1000) × 0.5 + 40 + 20 = 340 枚碎片/月
V4 標準：(1,000,000 人 / 1000) × 0.25 + 80 + 30 = 360 枚碎片/月
V5 標準：(1,200,000 人 / 1000) × 0.2 + 120 + 40 = 400 枚碎片/月
```

### 暴露風險動態計算

```
當前暴露概率 = 基礎風險 + (使用頻率 × 累積係數) - (靜默時間 × 衰減係數)

基礎風險：5%
累積係數：每次使用 +5% ~ +10%（取決於使用複雜度）
衰減係數：每 24 小時 -50%（靜默期間無使用）
```
