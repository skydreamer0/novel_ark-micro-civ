### 污染類型與處理方式

| 污染類型 | 處理方式 |
|----------|----------|
| 流程殘留（L3-L4 `工作流：Write Chapter`） | 全刪 |
| 企劃語混入（「真正的爽感是…」「本章目標」） | 全刪，改成場景承載 |
| 模板句重複（同一句連續數十次） | 111-120不留局部修補，直接重寫 |
| 交付摘要（`本章模式B交付摘要`、條列式說明） | 全刪 |
| 跨章通用治理段（多章共用民生帳/三條釘段） | 刪除或壓縮成必要結果 |
| 章節功能互覆（不同章共享骨架） | 每章重建單一場景核心 |

### Agent 1：清理第101-110章

**範圍：** `01_NOVEL_CONTENT/04_第四卷_投放名冊/第101章_戰場開盤.md` ～ `第110章_十秒鐘.md`

**方式：** 逐章讀取，依審稿表精確行號刪除污染內容，保留主事件段落。

**每章處理步驟：**
1. 刪除 L3-L4 流程殘留
2. 刪除 L8-L12、L96-L103 企劃列點/交付摘要（依各章審稿表精確位置）
3. 刪除模板句重複區塊（L24-L92）
4. 刪除跨章通用治理段（L174-L198 附近）
5. 刪除模式B總結段落
6. 確認剩餘正文3000-4500字、事件完整、章末有乾淨鉤子
7. 依審稿表「建議修復方式」補強

### Phase 5：建立防混規則

新增或更新寫作檢查規則：
- 正文不得包含 `工作流：`、`模式B交付摘要`
- 正文不得出現「本章爽點」「本章目標」「三線推進」等企劃語
- 單章內同一句完整句子不得重複超過 2 次
- 章節結尾必須是場景鉤子或狀態變化
- 每次生成後先跑污染檢查
