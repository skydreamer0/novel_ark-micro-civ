# 第101～120章混稿修復 Roadmap

> 建立日期：2026-05-16
> 範圍：第四卷《投放名冊》第101～120章
> 目標：先停止混稿擴散，再分批修復正文、狀態錨點與續寫流程。

---

## 1. 問題判定

目前問題不是單章品質波動，而是後段生成流程出現系統性混稿。已觀察到四類污染：

1. **流程殘留進正文**
   - 多章保留 `工作流：Write Chapter｜模式B：正文生成`。
   - 多章結尾保留 `本章模式B交付摘要`。
   - 這些內容應屬於生成紀錄，不應進入小說正文。

2. **模板句大量重複**
   - 第111～120章多次重複「冷光從破裂的導光板上……」。
   - 同一句在單章內連續出現數十次，表示生成迴圈失控。

3. **企劃語混入正文**
   - 例如「這一章真正的爽點不是……」。
   - 這類句子屬於審稿分析或章節意圖，不是角色、場景或敘事。

4. **章節功能互相覆蓋**
   - 第111～120章的事件差異變小，常用同一組「民生帳、證據、重建、制度反擊」語彙收束。
   - 章名不同，但正文骨架相近，導致讀感像同一章換標題。

---

## 2. 修復原則

1. **不急著續寫第121章**
   - 第五卷開局必須接在乾淨的第120章狀態上。
   - 若直接續寫，會把第四卷後段的錯誤狀態帶進第五卷。

2. **先分類，再重寫**
   - 不直接整批覆蓋第101～120章。
   - 每章先標記為「可保留」「需大修」「建議重寫」。

3. **保留可用事件，重寫敘事承載**
   - 若章節核心事件可用，保留事件與結果。
   - 刪除模板句、交付摘要、企劃語，重寫場景、動作、對話與反派反應。

4. **第四卷收束要回到本書主承諾**
   - 爽點核心應是「拆制度、拿證據、建立可持續反擊機制」。
   - 避免寫成純粹「滿級神裝、零戰損、全員臣服」。

## 3. 章節分級初稿

### A. 優先檢查：第101～110章

暫定狀態：**需逐章審核，可能可保留主事件**

已知問題：
- 多章帶有流程抬頭與交付摘要。
- 需確認正文是否只是格式污染，還是已經出現章節互混。

檢查方式：
- 刪除生成流程殘留後，讀章節是否仍能獨立成立。
- 對照第四卷章回落地卡，確認每章是否完成自己的單一功能。

### B. 高危區：第111～120章

暫定狀態：**建議大修或重寫**

已知問題：
- 模板句密集重複。
- 多章結構高度雷同。
- 正文混入審稿語。
- 卷末狀態不夠可靠，會影響第五卷開局。

處理方向：
- 先保留每章「不可逆事件」。
- 每章重建一個明確場景核心。
- 每章只承擔一個主要功能，不再用同一套總結段落收束。

## 4. 建議修復順序

### Phase 1：建立審稿清單

輸出一張第101～120章審稿表，每章記錄：
- 章名
- 原定章節功能
- 實際正文完成度
- 污染類型
- 可保留事件
- 需刪除內容
- 修復判定：可保留 / 需大修 / 建議重寫

完成標準：
- 能清楚知道每章要修多少，不靠感覺判斷。

### Phase 2：清理第101～110章表層污染

工作項目：
- 移除流程抬頭。
- 移除交付摘要。
- 檢查是否有企劃語。
- 檢查章節是否仍具備獨立事件推進。

完成標準：
- 每章讀起來像小說正文，而非工作流輸出。

### Phase 3：重建第111～120章

建議每章先定一個唯一功能：

| 章節 | 建議核心功能 |
|---|---|
| 111 | 外部接應戰，蘇岚主導火力與資訊壓制 |
| 112 | 內部文明信仰爆發，但引出治理風險 |
| 113 | 反向投放完成，奪取名冊與核心資源 |
| 114 | 撤離戰，不是無腦凱旋，而是帶著戰果突圍 |
| 115 | 戰後財報與創傷盤點，爽感與代價同場 |
| 116 | 分贓與重建，確立自治交換條件 |
| 117 | 無加購短暫黎明，民生改善但藏反制探針 |
| 118 | 非對等不可獻祭，內部文明立法與裂痕 |
| 119 | 結算與升級，名冊轉化為預警網但有暴露代價 |
| 120 | 卷末公聽/戰時治理核心成立，導向黑箱斷流 |

完成標準：
- 每章有獨立事件、獨立場景、獨立情緒轉折。
- 不再依賴同一段「冷光」「民生帳」「這章爽點」收束。

### Phase 4：同步資料庫狀態

需更新：
- `02_PROJECT_DATABASE/02_進度追蹤/目前進度.md`
- `02_PROJECT_DATABASE/00_核心設定/11_伏筆追蹤.md`
- 必要時補一份「第120章卷末狀態錨點」。

完成標準：
- 續寫第121章時，只會讀到第120章後的正確狀態。
- 不再引用第42章作為唯一進度參照。

### Phase 5：建立續寫防混規則

新增或更新寫作檢查規則：
- 正文不得包含 `工作流：`、`模式B交付摘要`。
- 正文不得出現「本章爽點」「本章目標」「三線推進」等企劃語。
- 單章內同一句完整句子不得重複超過 2 次。
- 章節結尾必須是場景鉤子或狀態變化，不得是交付說明。
- 每次生成後先跑污染檢查，再進入下一章。

完成標準：
- 第五卷續寫前有硬性防呆，不讓同類混稿重演。
