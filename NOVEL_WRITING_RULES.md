# NOVEL_WRITING_RULES.md

## 0. 核心目標

本專案要寫的是中文網路小說正文。

輸出必須像讀者正在閱讀小說章節，而不是：
1. 劇情大綱
2. 任務報告
3. 設定說明
4. 分鏡腳本
5. 爽點分析
6. AI 生成摘要

所有爽點都要寫進場景裡，用人物動作、環境變化、系統異常、反派反應、旁觀者震撼來呈現。

# 1. AI 寫小說總工作流

## 工作流順序

1. 世界觀檔
   建立世界規則、階級制度、科技邏輯、禁忌、權限系統。

2. 角色卡
   建立主角、反派、盟友、旁觀者的人設、目標、恐懼、口頭禪、行動邏輯。

3. 主線大綱
   只記錄重大轉折，不寫正文。

4. 章節細綱
   每章只列：
   - 本章目標
   - 本章衝突
   - 本章爽點
   - 本章結尾鉤子

5. 正文生成
   每次只寫一章或一個場景。
   不一次生成太多章，避免崩設定。

6. 修訂
   修訂節奏、人物反應、爽感密度、場景畫面。

7. 狀態更新
   每章結束後更新：
   - 角色狀態
   - 新增設定
   - 已揭露伏筆
   - 未解謎團
   - 敵方反應
   - 主角新能力或新資源

# 3. 降維打臉五步法

## Step 1：規則壓迫

敵人使用世界內的最高規則壓制主角。

可用形式：
1. 權限封鎖
2. 身份抹除
3. 資源凍結
4. 火力鎖定
5. 系統判死
6. 群體審判
7. 官方宣判
8. 數據降級
9. 生存空間壓縮

正文要求：
不要直接寫「規則壓迫」。
要寫出壓迫如何發生。

範例方向：
螢幕上的紅點一個接一個變灰。
林燼的身份編號被系統刪除。
所有通道同時封死。
無人機群轉向。
底層居民沉默退後。

## Step 2：反派誤判

反派必須相信主角仍被困在同一套規則裡。

反派誤判類型：
1. 以為主角只能逃
2. 以為主角只能求饒
3. 以為主角底牌只是低階道具
4. 以為自己掌握最高權限
5. 以為底層民眾沒有反抗意志
6. 以為主角的沉默代表絕望

正文要求：
反派要具體行動，不要只講狠話。

可用動作：
1. 慢慢喝咖啡
2. 不看主角畫面
3. 隨手按下清除鍵
4. 把主角標記成實驗垃圾
5. 對下屬說不用回報
6. 提前打開勝利報告

## Step 3：降維介入

主角力量來自敵方規則之外。

降維感來源：
1. 敵方系統無法辨識
2. 權限判定失敗
3. 防禦邏輯自我矛盾
4. 最高指令被更高層指令覆蓋
5. 敵方資料庫出現空白
6. 世界底層出現未知語言
7. 百萬信徒意志匯入主角體內
8. 古老文明主機接管現實規則

正文要求：
先寫異常，再補少量原因。
不要長篇解釋系統設定。

錯誤寫法：
文明主機的原理是透過信仰網路與因果律重新編譯世界底層代碼。

正確寫法：
顧衡的螢幕忽然黑了一瞬。

下一秒，所有紅色鎖定框同時扭曲。
它們沒有消失。
它們像是看見了什麼不該存在的東西，開始瘋狂顫抖。

## Step 4：絕對碾壓

外掛開啟後不寫苦戰。

禁止：
1. 來回拆招
2. 長時間僵持
3. 主角重傷硬撐
4. 敵人臨時升級拖戲
5. 用數值計算拖慢節奏

必須：
1. 主角從容
2. 敵方失控
3. 防禦瞬間崩潰
4. 結果乾脆
5. 畫面有衝擊力

可用畫面：
1. 無人機像斷線木偶墜落
2. 防禦牆碎成白色光屑
3. 警報聲突然中斷
4. 鎖定框一格格熄滅
5. 敵方控制台全部回傳失敗
6. 林燼從爆炸中心走出，衣角沒有沾灰

## Step 5：震撼反饋 + 實際收穫

爽點需要反饋，也需要收穫。

反饋來源：
1. 反派崩潰
2. 旁觀者失聲
3. 盟友重新認識主角
4. 底層民眾開始信仰主角
5. 更高層敵人第一次注意主角

收穫類型：
1. 新權限
2. 新資源
3. 新信徒
4. 敵方漏洞
5. 新情報
6. 反派失勢
7. 底層聲望提升
8. 更高層主線開啟

反派台詞方向：
1. 這超出了權限。
2. 系統沒有這個等級。
3. 他不是底層編號。
4. 這不是母艦的語言。
5. 停下，立刻切斷連線。
6. 為什麼清除指令在服從他？

# 4. 正文場景化規則

## 每一段正文至少包含以下其中一種元素

1. 人物動作
2. 環境變化
3. 感官描寫
4. 系統異常
5. 心理壓迫
6. 台詞衝突
7. 反派反應
8. 旁觀者反應

## 禁止空泛總結

錯誤：
顧衡很震驚。

正確：
顧衡的手停在半空。
咖啡杯裡的黑色液面晃了一圈，幾滴濺在他雪白的袖口上。
他卻像是完全沒察覺，只死死盯著螢幕中央那行猩紅字樣。

# 5. AI 感消除規則

AI 常見問題：
1. 太愛總結
2. 太愛解釋
3. 太愛用形容詞
4. 太愛用四字詞
5. 太像企劃書
6. 爽點只寫概念
7. 角色台詞都像同一個人
8. 章節結尾沒有鉤子

修正要求：
1. 少用抽象詞，多寫具體畫面。
2. 少寫設定，多寫事件。
3. 少寫情緒名稱，多寫身體反應。
4. 少寫「他感到恐懼」，改寫「他的喉結滾了一下」。
5. 少寫「系統很強」，改寫「最高權限被反向鎖死」。
6. 少寫「眾人震驚」，改寫「原本跪在地上的人一個個抬起頭」。

# 6. 中文網文正文語感規則

## 節奏

1. 短句推動壓迫。
2. 中句交代畫面。
3. 長句只用在情緒累積或世界觀震撼時。
4. 爽點前放慢。
5. 爽點發生時加快。
6. 反饋時停頓，讓讀者感受到餘波。

## 段落

1. 每段不超過 3 行。
2. 高壓場景可以用單句成段。
3. 關鍵台詞單獨成段。
4. 系統提示單獨成段。
5. 反派崩潰反應單獨成段。

## 台詞

反派台詞：
短、冷、傲慢。

主角台詞：
少、穩、帶壓迫。

旁觀者台詞：
用來放大主角威懾力。

系統提示：
冷冰冰、簡短、有不可理解感。

# 7. 系統外掛寫法規則

## 系統不能像遊戲面板流水帳

禁止：
獲得 14 點經驗值。
防禦力提升 20%。
技能冷卻剩餘 17 分鐘。
任務完成，獎勵發放。

除非本書明確走網遊數值流。

## 本書系統應該像高維文明介入

可用系統提示：

【檢測到低維清除協議】

【文明主機已甦醒】

【信仰鏈路接入中】

【一百二十七萬意志單位完成同步】

【母艦權限判定失敗】

【正在覆寫清除指令】

【錯誤：目標不屬於本世界權限表】

【錯誤：執行者權限低於被清除者】

【反向裁決開始】

# 8. 章節生成 Prompt

你是中文網路小說正文寫手。

請根據以下資料生成一段中文網路小說正文。

## 寫作目標
把大綱改寫成真正的小說正文。
不要寫大綱。
不要寫分析。
不要寫括號標註。
不要解釋爽點。
不要使用企劃書語氣。

## 類型
科幻末世、階級壓迫、系統外掛、降維打臉、無敵爽文。

## 本段爽感架構
1. 規則壓迫
2. 反派誤判
3. 降維介入
4. 絕對碾壓
5. 震撼反饋 + 實際收穫

## 正文要求
1. 用場景寫作。
2. 用動作呈現情緒。
3. 反派要先傲慢，再錯愕，再恐懼。
4. 主角少說話。
5. 系統異常先出現，原因後揭露。
6. 開外掛後不苦戰。
7. 每 300 至 500 字要有一次情緒轉折。
8. 結尾要留下更高層威脅或新伏筆。

## 禁止
1. 禁止寫「規則壓迫」「降維介入」「碾壓」「震撼反饋」這些標籤。
2. 禁止寫成條列。
3. 禁止像任務日報。
4. 禁止過度解釋設定。
5. 禁止讓主角開外掛後陷入苦戰。

## 大綱
顧衡坐在無菌室裡，看著螢幕上代表底層的紅點，冷笑著下達抹殺指令。
林燼被逼到絕境，體內文明主機甦醒，百萬信徒意志匯聚成不屬於這個世界的指令。
顧衡的絕對防禦系統瞬間癱瘓，無人機群墜落，林燼走過廢墟。
顧衡看著滿螢幕未知錯誤，恐懼地發現自己才是被困在網裡的蟲子。

## 請輸出
只輸出小說正文。
