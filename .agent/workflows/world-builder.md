# 世界觀擴張工作流 (World Builder)

此工作流用於在現有的「賽博母艦」框架下，定義新的細節（如：特定艙段、組織、技術或裂縫類型）。

## 執行步驟

### Step 1: 讀取核心設定
// turbo
1. 讀取 `docs/03_世界觀/01_母艦與都市結構.md`，確認母艦結構與生存方程式。
// turbo
2. 讀取 `docs/03_世界觀/03_資源與轉化規則.md`，確認資源類型與轉化流程。
// turbo
3. 讀取 `docs/03_世界觀/04_社會秩序與壓迫.md`，確認稅制與經濟數值。
// turbo
4. 讀取 `docs/02_核心設定/03_力量與文明系統.md`，確認「雙層煉化權限」與「文明主機」邏輯。
// turbo
4.1. 讀取 `docs/03_世界觀/06_熵潮與存在級威脅.md`，確認熵化邏輯與規則密度設定。
// turbo
4.2. 若涉及艙段或社會組織，讀取 `docs/03_世界觀/07_其他艙段與平行社會.md` 及 `08_都市信仰與意識形態.md`。

### Step 2: 定義新元素
5. 使用以下結構描述新物件：
   - **名稱**：（例如：第 13 號貧民窟「鐵鏽帶」）
   - **視覺特徵**：（陰冷、霓虹燈閃爍、機油味）
   - **壓迫機制**：（該地段受哪個公司控制？稅收如何？）
   - **機遇點**：（主角能在這裡拾取什麼外域殘骸？）
6. 若新增的是**裂縫類型**，按照 `docs/03_世界觀/02_場景與生態危機.md` 的裂縫分類標準，先指定五大類之一，再加 1～2 個詞綴。
7. 若新增的是**勢力**，需定義：收入來源、壟斷手段、對主角態度（參照 `docs/02_核心設定/02_角色與陣營/02_陣營與敵對勢力.md` 的格式）。

### Step 3: 衝突檢查
8. 檢查新設定是否破壞了以下硬約束：
   - 「母艦宇宙正在衰亡」的大背景（必須未違背熵潮與規則密度流失的邏輯）
   - 母艦生存方程式的三條硬約束（能源、人口、污染）
   - 主角的核心差異（雙層煉化 + 唯一反熵的文明主機）
9. 確認沒有與 `docs/03_世界觀/05_專有名詞辭典.md` 中已定義的名詞衝突。

### Step 4: 更新文檔
10. 將確認的新設定寫入對應的世界觀子文件：
    - 場景 → `docs/03_世界觀/02_場景與生態危機.md`
    - 母艦結構 → `docs/03_世界觀/01_母艦與都市結構.md`
    - 資源 → `docs/03_世界觀/03_資源與轉化規則.md`
    - 制度 → `docs/03_世界觀/04_社會秩序與壓迫.md`
11. 同步更新 `docs/03_世界觀/05_專有名詞辭典.md`，為每個新名詞建立條目。
12. 若涉及角色或勢力，同步更新 `docs/02_核心設定/02_角色與陣營/` 下的對應文件。
