# 伏筆追蹤更新 (Update Foreshadowing)

讀取指定章節，掃描新埋的伏筆與已回收的伏筆，更新 `docs/00_企劃大綱/07_伏筆追蹤表.md`。

## 執行步驟

### Step 1: 讀取追蹤表
// turbo
1. 讀取 `docs/00_企劃大綱/07_伏筆追蹤表.md`，載入所有伏筆的當前狀態。

### Step 2: 掃描章節
// turbo
2. 讀取使用者指定的章節範圍（如：第43～50章）。
3. 逐章掃描以下類型的伏筆信號：
   - **回收信號**：上一次埋設的線索、人物、事件被再次提及或解決
   - **新埋信號**：出現新的未解問題、懸念、暗示或情報碎片
   - **加深信號**：已有伏筆被進一步強化但未完全回收

### Step 3: 更新狀態
4. 對每條已有伏筆判斷狀態變化：
   - 🔴 → 🟡：本章有提及但未完全回收
   - 🔴 → 🟢：本章已完整回收
   - 🟡 → 🟢：本章完成最終回收
5. 對新埋的伏筆建立新條目：
   - 伏筆描述
   - 埋設章節
   - 分類（真相線 / 角色線 / 制度線 / 力量線）
   - 狀態設為 🔴

### Step 4: 產出報告並更新
6. 產出變更摘要：
   - 新增了幾條伏筆
   - 回收了幾條伏筆
   - 當前 🔴 / 🟡 / 🟢 統計
7. **呈給使用者確認**。
8. 確認後寫入 `docs/00_企劃大綱/07_伏筆追蹤表.md`。

### Step 5: 超時警告
9. 檢查是否有 🔴 伏筆已超過 8 章未回收（參照 `NOVEL_WRITING_RULES.md` §4.8：前期伏筆原則上在 3～8 章內要有第一次回收或加深）。
10. 若有，特別標記並建議在近期章節安排回收。
