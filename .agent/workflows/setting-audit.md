# 設定一致性審查 (Setting Audit)

掃描指定章節範圍，與設定文檔交叉比對，找出潛在的設定矛盾、角色行為偏移或數值不一致。

## 執行步驟

### Step 1: 載入設定基準
// turbo
1. 讀取以下設定文件作為比對基準：
   - `docs/02_核心設定/01_核心設定與賣點.md`
   - `docs/02_核心設定/02_角色與陣營/01_主角成長軌跡.md`
   - `docs/02_核心設定/02_角色與陣營/02_陣營與敵對勢力.md`
   - `docs/02_核心設定/03_力量與文明系統.md`
   - `docs/03_世界觀/` 底下所有設定文件（包括資源、社會、專有名詞、熵潮、艙段與信仰等，視章節屬性讀取相關篇章）

### Step 2: 掃描章節
// turbo
2. 讀取使用者指定的章節範圍。
3. 逐章檢查以下維度：

#### 角色行為一致性
- 林燼是否符合當前卷的行為邏輯（利益導向 / 契約精神 / 斬草除根）？
- 是否出現違反 `01_主角成長軌跡.md` 規定的聖母行為或無動機善意？
- 盟友是否符合各自的致命弱點與價值衝突點？

#### 力量系統一致性
- 反哺能力是否超出當前文明時代的可用範圍？
- 是否出現無代價的免費收穫？
- 規則碎片、存活點等數值是否與設定文件的基準範圍一致？

#### 世界觀一致性
- 稅制數值是否與 `04_社會秩序與壓迫.md` 的基準一致？
- 是否出現高層、企業或審計系統被降智的情節？
- 專有名詞用法是否與辭典一致？

#### 名稱一致性
- 角色名是否前後統一（無混用）？
- 地名、組織名是否前後統一？

### Step 3: 產出審查報告
4. 將發現的問題分為三級：
   - 🔴 **矛盾**：直接違反設定的硬性錯誤
   - 🟡 **偏差**：不算錯誤但偏離設定精神
   - 🟢 **建議**：可以更好的地方
5. 每個問題附上：
   - 出處（章節 + 行號/段落）
   - 違反的設定條目
   - 建議修正方式

### Step 4: 修正建議
6. 對 🔴 矛盾項，提供具體修改文字建議。
7. 對 🟡 偏差項，提供調整方向。
8. **呈給使用者確認**——使用者決定是否採納修正。
