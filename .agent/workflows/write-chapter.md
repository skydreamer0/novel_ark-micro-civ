# 端到端寫章節 (Write Chapter)

一條龍完成新章節：脈絡對齊 → 大綱確認 → 正文撰寫 → 品質審查 → 文檔更新。

## 前置條件
- 使用者需指定要寫的章節編號（如：第43章）
- 使用者可選擇性提供章節大綱或關鍵事件

## 執行步驟

### Phase 1: 脈絡對齊（自動執行）
// turbo
1. 讀取上一章的完整內容，重點關注最後 500 字的懸念與未結事件。
// turbo
2. 讀取 `docs/00_企劃大綱/03_第一卷大綱.md` 對應章節的規劃。
// turbo
3. 讀取 `docs/00_企劃大綱/07_伏筆追蹤表.md`，列出本章應處理的 🔴 伏筆。
// turbo
4. 讀取 `docs/02_核心設定/03_力量與文明系統.md`，確認主角當前能力階段。
// turbo
5. 讀取 `docs/02_核心設定/02_角色與陣營/01_主角成長軌跡.md`，確認當前卷的主角行為原則。
// turbo
6. 讀取 `NOVEL_WRITING_RULES.md`，確認所有硬性規格。
// turbo
7. 視本章劇情需要，讀取 `docs/03_世界觀/06_熵潮與存在級威脅.md`、`07_其他艙段與平行社會.md` 或 `08_都市信仰與意識形態.md` 獲取環境與組織設定。

### Phase 2: 章節規劃（需確認）
8. 基於以上脈絡，產出章節大綱：
   - 本章目標衝突
   - 三線推進計畫
   - 3-5 個關鍵場景
   - 章末鉤子
   - 需回收的伏筆
8. **呈給使用者確認**——使用者可調整後再進入 Phase 3。

### Phase 3: 正文撰寫
9. 按照 `novel-writer` Skill（`.agent/skills/novel-writer/SKILL.md`）的規範撰寫正文：
   - **開頭 (0-500字)**：切入危機，回應上一章懸念
   - **中段**：阻力升級，主角主動行動
   - **中後段**：至少一個不可逆變化
   - **結尾**：留下下一章動力
10. 檢查字數，確保 **≥ 3000 字**。
11. 將正文寫入 `docs/01_正文/第XX章_章名.md`。

### Phase 4: 自動審查
12. 對照 `NOVEL_WRITING_RULES.md` §7 完稿檢查清單逐項確認（等同執行 `/chapter-qa`）。
13. 若發現問題，自動修正後重新確認。

### Phase 5: 文檔同步
14. 更新 `docs/00_企劃大綱/07_伏筆追蹤表.md`：
    - 將本章回收的伏筆標記為 🟢
    - 新埋的伏筆加入 🔴
// turbo
15. 掃描本章正文，找出未在 `docs/03_世界觀/05_專有名詞辭典.md` 收錄的新名詞，提示使用者是否新增。
