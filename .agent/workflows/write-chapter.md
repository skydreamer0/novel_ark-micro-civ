# 端到端寫章節 (Write Chapter)

一條龍完成新章節：脈絡對齊 → 大綱確認 → 正文撰寫 → 品質審查 → 文檔更新。

## 前置條件
- 使用者需指定要寫的章節編號（如：第43章）
- 使用者可選擇性提供章節大綱或關鍵事件

## 執行步驟

### Phase 1: 脈絡對齊（自動執行）
// turbo
1. 讀取上一章的完整內容，重點關注最後 500 字的懸念與未結事件。

   - 1～30 章 → `01_第一卷_艙底種火.md`
   - 31～60 章 → `02_第二卷_裂縫稅戰.md`
// turbo
3. 讀取 `docs/00_企劃大綱/02_寫作與追蹤/01_伏筆追蹤表.md`，列出本章應處理的 🔴 伏筆。

7. 視本章劇情需要，讀取 `docs/03_世界觀/06_熵潮與存在級威脅.md`、`07_其他艙段與平行社會.md` 或 `08_都市信仰與意識形態.md` 獲取環境與組織設定。

### Phase 1.5: 連貫性護欄（自動檢查）
8. 確認以下凍結原則不被違反：
   - **不否定既有動機**：角色在前文做過的決策，只能補充背景，不能推翻。
   - **不新增逆天設定打臉前文**：新能力、新勢力必須以「前文已出現線索」作為入口。
   - **不改既有節奏定位**：維持「高壓求生＋制度反打」節奏，不可切換成慢熱日常。
   - **禁止打破第四壁**：敘事與對話必須 100% 沉浸於世界觀，絕不能提及「作者」、「讀者」、「小說」或做出跳脫角色當前認知的後設吐槽。
9. 角色行為約束：
   - 林燼：利益導向不變，不寫成突然聖母；可增加「長線選擇焦慮」。
   - 技術位盟友：功能不只開門駭入，要承擔「抹痕失敗」風險。
   - 戰鬥位盟友：不能只當肉盾，每卷需有一次主動決策造成翻盤或災難。
   - 反派層級遞進：幫派 → 企業 → 審計，避免同質化追兵。

### Phase 2: 章節規劃（需確認）
8. 基於以上脈絡，產出章節大綱：
   - 本章目標衝突
   - 三線推進計畫
   - 3-5 個關鍵場景
   - 章末鉤子
   - 需回收的伏筆
9. **呈給使用者確認**——使用者可調整後再進入 Phase 3。

### Phase 3: 正文撰寫
10. 按照 `novel-writer` Skill（`.agent/skills/novel-writer/SKILL.md`）的規範撰寫正文：
    - **開頭 (0-500字) [過渡護欄]**：嚴格禁止時間跳躍或總結式開場。必須以「延續性動作」（如未落下的手、迴盪的腳步聲、剛脫口而出的對話尾音）作為切入點，確保物理空間與動作在章節間的無縫連貫，再順勢帶出當前危機。
    - **中段**：阻力升級，主角主動行動
    - **中後段**：至少一個不可逆變化
    - **結尾**：留下下一章動力
11. 檢查字數，確保 **≥ 3000 字**。
12. 將正文寫入 `docs/01_正文/第XX章_章名.md`。

### Phase 4: 自動審查
13. 對照 `NOVEL_WRITING_RULES.md` §7 完稿檢查清單逐項確認（等同執行 `/chapter-qa`）。
14. 若發現問題，自動修正後重新確認。

### Phase 5: 文檔同步
15. 更新 `docs/00_企劃大綱/02_寫作與追蹤/01_伏筆追蹤表.md`：
    - 將本章回收的伏筆標記為 🟢
    - 新埋的伏筆加入 🔴
// turbo
