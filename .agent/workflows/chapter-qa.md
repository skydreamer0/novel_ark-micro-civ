# 章節品質控管與審批 (Chapter QA)

本工作流為小說專用的「品管 (QA)」機制。當一章完成後，系統將喚醒內部的 **6 位專家** 對本章進行嚴格審查，確保符合《末日母艦》的硬性寫作規範，杜絕水字數、降智與拼接感。

## 審查執行步驟

### Step 1: 讀取待審查章節與參照物
// turbo
1. 讀取使用者指定的章節檔案（`docs/01_正文/第XX章_*.md`）。
// turbo
2. 讀取上一章的最後 500 字，檢查連貫性。
// turbo
3. 讀取 `NOVEL_WRITING_RULES.md` 的完稿檢查清單與世界觀大綱以作比對。

### Step 2: 基礎規格品管 (門檻檢查)
1. **字數檢核**：統計正文字數，確認是否達到 **3000 字** 到 **4500 字** 的標準。排除空白行與章名。低於會被直接標記為「字數不足」。
2. **開場檢核**：確認前 500 字內是否立刻出現「危機、任務、交易、追殺、衝突或異常」，拒絕冗長鋪墊。

### Step 3: 六位專家聯合會審 (核心品管)
系統由內部 6 位專家輪流提出審核意見，並整合結果：

| 評審專家 | 核心審核項目 (一票否決權) |
| --- | --- |
| **1. 世界觀策士** | 敵方/高層是否降智？是否存在為了爽而製造的低級失誤？世界配置是否被破壞？說明是否百科化？ |
| **2. 節奏編劇** | 每章是否具備：明確目標、真實阻力、狀態改變、指向下一步的壓力？章尾鉤子是否足夠強烈？ |
| **3. 角色操盤手** | 利益交換與角色動機是否成立？是否違背人物既有行為邏輯 (比如突然無私聖母)？ |
| **4. 爽點工程師** | 是否至少推進了 3 條主線中的 2 條？升級/收穫是否付出了**代價**？文明線是否有反哺外部主線，還是自顧自發展脫節？ |
| **5. 文風修補師** | 是否有明顯的「拼接感」？資訊嵌入是否生硬？有沒有使用摘要式灌水空轉情節？ |
| **6. 伏筆審核員** | 必須發生至少 1 個「不可逆變化」。新伏筆是否突兀？舊伏筆回收是否合理？ |

### Step 4: 伏筆與辭典連動檢查
- 比對 `01_伏筆追蹤表.md`，確認本章是否有可以標注為回收 🟢 的項目。
- 掃描全文捕捉是否出現未被收錄於 `05_專有名詞辭典.md` 的新名詞，建議補入。

### Step 5: 產出綜合品管報告與評級
整合 6 位專家的審查意見，給出本章的總體品質評級與修改清單：

- **🟢 準備發布 (Ready)**：所有專家亮綠燈，字數滿標，無退稿缺失。
- **🟡 需微調 (Minor Fix)**：1-2 位專家提出警告（例如：代價不夠痛、結尾鉤子稍弱），建議修改後發布。
- **🔴 需重寫 (Rewrite)**：違反核心禁忌（例如：連續重複已知設定、敵方降智、字數嚴重不足、未推進任何主線），專家退回並附上必須打掉重練的段落與原因。
