# 章節品質控管與審批 (Chapter QA)

本工作流為小說專用的「品管 (QA)」機制。當一章完成後，系統將喚醒內部的 **6 位專家** 對本章進行嚴格審查，確保符合《末日母艦》的硬性寫作規範，杜絕水字數、降智與拼接感。

## 審查執行步驟

### Step 1: 讀取待審查章節與參照物
// turbo
1. 讀取使用者指定的章節檔案（`01_NOVEL_CONTENT/0X_第X卷/第XXX章_*.md`）。
// turbo
2. 讀取上一章的最後 500 字，檢查連貫性。
// turbo
3. 讀取 `02_PROJECT_DATABASE/RULES.md`（寫作規範與完稿檢查清單）。
// turbo
4. 讀取 `02_PROJECT_DATABASE/CANON_world.md`（世界觀大綱，用於比對設定一致性）。

### Step 2: 基礎規格品管 (門檻檢查)
1. **字數檢核**：統計正文字數，確認是否達到 **3000 字** 到 **4500 字** 的標準。排除空白行與章名。低於會被直接標記為「字數不足」。
2. **開場檢核**：確認前 500 字內是否立刻出現「危機、任務、交易、追殺、衝突或異常」，拒絕冗長鋪墊。

### Step 3: 六位專家聯合會審 (核心品管)
依據 `02_PROJECT_DATABASE/RULES.md §7 專家小組` 的標準定義，由 6 位專家輪流提出審核意見並整合結果。

### Step 4: 伏筆與辭典連動檢查
- 比對 `02_PROJECT_DATABASE/STATE.md §6 開放伏筆`，確認本章是否有可以標注為回收 🟢 的項目。
- 掃描全文捕捉是否出現未被收錄於 `02_PROJECT_DATABASE/CANON_world.md §23 統一術語表` 的新名詞，建議補入。

### Step 5: 產出綜合品管報告與評級
整合 6 位專家的審查意見，依據 `09_expert_panel.md` 中的三級品質評級給出本章評級與修改清單：

- **🟢 準備發布 (Ready)**：所有專家亮綠燈，字數滿標，無退稿缺失。
- **🟡 需微調 (Minor Fix)**：1-2 位專家提出警告（例如：代價不夠痛、結尾鉤子稍弱），建議修改後發布。
- **🔴 需重寫 (Rewrite)**：違反核心禁忌（例如：連續重複已知設定、敵方降智、字數嚴重不足、未推進任何主線），專家退回並附上必須打掉重練的段落與原因。
