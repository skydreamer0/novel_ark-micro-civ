# 章節節奏設計工作流 (Chapter Planner)

此工作流確保每一章節都符合「爽文」節奏，並同時推進戰力、文明與陰謀三條線。

## 執行步驟

### Step 1: 讀取上下文
// turbo
1. 讀取上一章結尾（`docs/01_正文/` 中最新章節的最後 1000 字），不僅找出**未完的危機與懸念**，更要準確捕捉**最後一刻角色的物理位置、姿態、環境細節與核心情緒**。
// turbo
2. 根據章節編號判斷所屬卷數，讀取 `docs/00_企劃大綱/01_分卷大綱/` 中對應卷大綱的 §7 章回落地卡，確認本章的「目標衝突」、「三線推進」與「章末鉤子」。
   - 1～30 章 → `01_第一卷_艙底種火.md`
   - 31～60 章 → `02_第二卷_裂縫稅戰.md`
// turbo
3. 讀取 `docs/00_企劃大綱/02_寫作與追蹤/01_伏筆追蹤表.md`，確認是否有 🔴 待回收的伏筆應在本章處理。
// turbo
4. 讀取 `docs/02_核心設定/03_力量與文明系統.md`，確認主角當前力量階段與可用反哺。
// turbo
5. 若劇情涉及高層制度、跨艙段或外域，讀取 `docs/03_世界觀/06_熵潮與存在級威脅.md`、`07_其他艙段與平行社會.md` 或 `08_都市信仰與意識形態.md` 作為當前環境壓力的底層依據。

### Step 2: 設定章節目標
6. 確定本章要完成的「三線進度」：
   - **個人戰力**：有新的裝備/數據提升嗎？有風險/代價嗎？
   - **文明成長**：內部文明有沒有新的演化/信仰反饋/制度事件？
   - **陰謀線**：是否揭露了一點高層的秘密或制度壓迫？
7. 確認本章是否需要回收伏筆（參照追蹤表中 🔴 項目）。
8. 確認本章的財務壓力節點（參照 `docs/00_企劃大綱/02_寫作與追蹤/02_財務壓力節點.md`）。

### Step 3: 三段式結構設計
9. 設計 **開頭 (The Hook) - 無縫承接要求**：
   - 拒絕時間軸跳躍，必須「緊接著上一章的最後一個動作/對話/畫面」開始。
   - 延續上一幕的物理狀態（姿勢、場景氣氛、正在發生的動作）。
   - 在前 500 字內帶入當前問題、任務、危機或衝突的延燒，避免使用回顧性或總結性旁白。
10. 設計 **中段 (Conflict)**：
    - 主角利用「維修工」技能或「主機截流」優勢翻盤
    - 阻力必須升級，不能一路平推
    - 加入至少一個賽博感官細節（義體過熱的滋滋聲、虛擬顯示幕的亂碼、機油與血腥味等）
11. 設計 **結尾 (Cliffhanger)**：
    - 必須出現代價、轉折、選擇或懸念
    - 為下一章埋下直接可承接的動力

### Step 4: 產出章節大綱
12. 條列式列出本章的 3-5 個關鍵場景，每個場景標註：
    - 場景目的（推進哪條線）
    - 關鍵衝突
    - 不可逆變化
13. 標註章末鉤子類型（生死倒數 / 新收益窗口 / 身分暴露風險）。
14. 將大綱寫入使用者指定的位置或直接呈現供確認。
