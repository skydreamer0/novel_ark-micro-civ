# 章節節奏設計工作流 (Chapter Planner)

此工作流為 **【模式A：章節規劃】**，確保每一章節都符合「爽文」節奏，並透過內部協作機制（六位專家）進行審查，同時推進戰力、文明與真相三條主線。

## 內置專家協作機制

> 詳見 `02_PROJECT_DATABASE/01_RULES/09_expert_panel.md`

**絕對禁止**：空泛熱血、無代價升級、長篇設定講解、明顯拼接感、新設定硬插、角色突然失智、只推一條線、沒有結果的衝突、為了拖字數而重複描述。

## 執行步驟

### Step 1: 讀取上下文
// turbo
1. 讀取上一章完整內容（`01_NOVEL_CONTENT/0X_第X卷/` 中最新章節的最後 1000 字），準確捕捉最後一刻角色的物理位置、姿態、環境細節與核心情緒，以便規劃無縫接軌。
// turbo
2. 根據章節編號判斷所屬卷數，讀取 `02_PROJECT_DATABASE/00_CORE/02_plot.md` 中對應卷的分卷大綱。
// turbo
3. 讀取 `02_PROJECT_DATABASE/00_CORE/05_foreshadowing.md`，確認是否有 🔴 待回收伏筆應在本章處理。
// turbo
4. 讀取 `02_PROJECT_DATABASE/02_DETAILS/02_power/主機版本能力詳表.md`，確認主角當前力量階段與可用反哺。
// turbo
5. 讀取 `02_PROJECT_DATABASE/00_CORE/07_conflict_arcs.md`，確認當前高張力弧線。

### Step 2: 確認本章目標 (使用者輸入或系統補全)
若使用者未給予完整指示，請依據上下文自行定義：
- **【本章目標】**：主角要完成什麼、拿到什麼、撐過什麼。
- **【本章必推主線】**：戰力線、文明線、真相線（至少兩條，文明線不可脫節）。
- **【本章主要衝突】**：壓力來源（例如：追殺、資源搶奪、配額壓迫等）。
- **【本章不可逆變化】**：結束後什麼東西必須已經改變不能回到原狀。
- **【本章結尾鉤子】**：下一章最直接能接什麼壓力或危機。

### Step 3: 產出【模式A】標準格式
綜合六位專家視角審查並產出最終方案，固定輸出以下 9 點：

1. 本章一句話核心
2. 四段式節奏規劃（需明確定義開局前500字的切入危機）
3. 本章推動的主線
4. 本章主要衝突與轉折
5. 本章代價與收穫（升級必須付出代價）
6. 本章不可逆變化
7. 本章章尾鉤子
8. 設定衝突檢查（策士與操盤手意見）
9. 可能產生拼接感的位置與修補方案（文風修補師意見）

### Step 4: 存放規劃結果
規劃結果呈給使用者確認後，作為下一步 `/write-chapter` 或 `/full-chapter-pipeline` Phase 2 的輸入。
