# 章節過渡優化工作流 (Optimize Transitions)

針對已經寫好的章節（如 $N-1$ 章與 $N$ 章），如果發現切換生硬、缺乏物理與情緒過渡，可使用此工作流重新補寫承接段落。

## 前置條件
- 使用者已指定需要優化的兩章（例如：`需優化 第50章 與 第51章 之間的過渡`）。

## 執行步驟

### Phase 1: 斷層診斷（自動讀取）
// turbo
1. 讀取 $N-1$ 章（上一章）的最後 1000 字，精確擷取「斷點快照」：
   - 角色當下確切的物理位置、姿勢與狀態。
   - 場景的光影、聲音、氣味（賽博感官細節）。
   - 對話停在誰的哪一句話，或動作停在哪個瞬間。
// turbo
2. 讀取 $N$ 章（目標章節）的前 1000 字，確認目前的「敘事起點」。
3. 進行斷層分析，診斷兩者之間缺失了什麼：
   - 時間跳轉？（漏掉了幾分鐘/幾小時的移動過程）
   - 空間突兀切換？（沒有寫怎麼走到下一個場景的）
   - 情緒/動作不連貫？（上一章還在激戰，下一章開頭就在喝茶總結）

### Phase 2: 過渡段重寫與擴寫（需確認）
4. 針對 $N$ 章的開頭段落（前 500~800 字），規劃重寫與擴寫方案：
   - **第一句必須是「無縫承接」**：銜接上一章最後一秒的動態（如對話尾音、環境餘波、未落下的手）。
   - **補足過渡細節**：將 $N-1$ 章結尾的情緒延續，安排合理的時間與物理空間轉換，自然引導到 $N$ 章原本要發生的主要事件。
   - **嚴禁總結式旁白**：不允許使用「就這樣，他們來到了…」之類的偷懶寫法，必須用具體的腳步、對話、喘息聲來推進。
5. 產出修改前後的 `Diff` 對比，或直接提供「建議替換的新開頭段落」。
6. **呈給使用者確認**——請使用者檢視過渡是否自然、是否消除跳躍感。

### Phase 3: 更新正文
7. 使用者確認後，依照修改建議，更新 $N$ 章 (`docs/01_正文/...`) 的開頭段落。
