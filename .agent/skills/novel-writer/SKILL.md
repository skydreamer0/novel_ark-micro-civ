# novel-writer 技能：撰寫/續寫《微光文明》章節

> 載入本技能後，AI 將依循以下指令完成寫作任務。

---

## 第一部分：專案概述

### 小說基本資訊
- **書名**：《微光文明》（英文：Mothership: Inner Micro-Civilization）
- **類型**：科幻末世 + 階級壓迫 + 文明主機外掛 + 稅戰經濟爽文
- **當前進度**：第1～120章完稿（共4卷），第5卷（121～150章）待續寫
- **一句話定位**：母艦 = 封閉作業系統，主角 = 入侵的病毒文明

### 核心隱喻
母艦「方舟」是一個封閉的物理作業系統。氧氣、空間、重力 = 硬體資源；存活點 = 低頻算力；信仰 = 高頻算力。主角林燼體內的「文明主機」是高維度新 OS，寄生在母艦中運行。

## 第二部分：寫作前必讀清單

在開始撰寫任何新章節前，**必須**按順序讀取以下文件：

1. `.ai/SUMMARY.md` — 全局記憶喚醒（全書一句話摘要 + 核心設定 + 進度）
2. `02_PROJECT_DATABASE/00_CORE/02_plot.md` — 當前狀態錨點（確認續寫起點）
3. `02_PROJECT_DATABASE/01_RULES/05_視角與資訊法.md` — 寫作技法（視角鎖定、資訊不對稱、感知寫作法）
4. `02_PROJECT_DATABASE/01_RULES/01_writing_rules.md` — 寫作規範（字數、節奏、章節連貫性）
5. `02_PROJECT_DATABASE/01_RULES/06_style_dna.md` — 風格喚醒（理想正文的感性範例）
6. `02_PROJECT_DATABASE/01_RULES/07_voice_bible.md` — 角色對話差異（防止角色聽起來像同一個人）
7. 目標卷**最後 3 章**正文 — 文風與連貫性確認

## 第三部分：寫作規則濃縮（11條精華）

1. **字數**：每章 ≥ 3000 字，常態 3200～4500
2. **節奏**：3 章一循環（行動 → 衝突 → 結算）
3. **視角**：每章鎖定單一感知主體，不跳入其他角色內心
4. **資訊差**：對話中角色不該解釋對方已知的事；保留「角色不說的話」作為張力
5. **禁止**：大綱感、企劃語、AI 味（連續成語、括號標籤、總結句）
6. **反派必備**：先傲慢 → 再錯愕 → 再恐懼
7. **主角**：少說話、開掛後不苦戰、每章要有收益
8. **系統提示**：像高維文明介入，非遊戲面板（如【權限判定中】，非【系統公告】）
9. **結尾**：必須留鉤子，不是結論
10. **禁止污染**：不得出現「工作流：」「模式B」「本章爽點」等流程殘留用語
11. **沉浸**：不得打破第四面牆，禁止後設用語

## 第五部分：品質閘門（寫完一章後強制檢查）

完成一章正文後，**必須**逐項通過以下六關檢查：

1. **視角是否鎖定？** — 全章是否只有一個感知主體？是否跳入了其他角色的內心？
2. **資訊差是否存在？** — 角色對話中有沒有解釋對方已知的事？有沒有保留「角色不說的話」？
3. **感知寫作是否到位？** — 場景是「看到、聽到、聞到、感覺到」還是「說明文」？
4. **場景化是否完成？** — 事件是否發生在具體的物理空間中？還是在真空中推進？
5. **結尾是否有鉤子？** — 最後一段是否留下懸念、轉折或不可逆變化？
6. **字數是否達標？** — ≥ 3000 字？若不達標，是否在 3200～4500 常態區間？

**任一關未過 → 退回修訂，不得視為完稿。**

## 第六部分：續寫工作流

完成一章正文並通過品質閘門後，執行以下後續步驟：

1. **更新 `02_PROJECT_DATABASE/00_CORE/02_plot.md` 的進度錨點**
   - 新增剛完成的章節到對應卷的轉折列表
   - 更新第120章狀態錨點為最新一章的狀態
2. **更新 `02_PROJECT_DATABASE/00_CORE/05_foreshadowing.md`**
   - 記錄新埋下的伏筆
   - 標記已回收的伏筆狀態
3. **更新 `.ai/SUMMARY.md`**
   - 更新進行中的敘事線描述
   - 更新第121章應處理事項（如有新變化）
   - 確認關鍵詞彙表無需增補
