# 小說正文撰寫助手 (Novel Writing Assistant)

這個 Skill 的主要目的是確保 AI 在協助創作、續寫或潤飾小說正文時，能夠嚴格遵守本作的硬式寫作規則（`NOVEL_WRITING_RULES.md`），維持小說的節奏、世界觀一致性與高張力。

## 🎯 核心原則

1. **零水字數與字數底線**：每一次的對話、動作或描述都必須為了推進劇情、交代代價或揭露世界觀而存在。**且每章正文內容必須達到或超過 3000 字**（不含章名、章號與空白行等非正文內容）。
2. **遵守硬性規範**：必須時刻以專案目錄下的 `NOVEL_WRITING_RULES.md` 作為最高指導原則。
3. **主角特殊性**：時刻記得主角的核心差異是「雙層煉化權限」與「文明主機」，並且主角的每一次升級與收穫必然伴隨風險、損耗或高層的監控。

## 📋 執行流程

當使用者要求你撰寫或修改章節正文時，請按照以下步驟進行：

### Step 1: 脈絡對齊
- 閱讀目標章節的上一章結尾，找出**未完的危機、懸念、代價或立刻要做的決定**。
- 確認使用者提供的本章大綱（包含：目標、阻力、結果、下一步）。
- 快速確認當前主線推動狀態（生存/戰力、文明/主機、母艦/陰謀），確保本章推動了至少兩條主線。

### Step 2: 正文撰寫 (The Actual Writing)
- **開頭 (0-500字)**：立刻切入當前任務、危機或衝突，直接回應上一章的懸念，不要慢吞吞地鋪墊日常。
- **中段**：讓阻力或衝突升級，主角必須採取主動行為，不得只靠被動挨打或被宣告。
- **中後段**：必須發生至少一個**不可逆的變化**（資源增減、關係變動、權限上升、情報揭露等）。設定說明必須依附在事件與動作之中。
- **結尾**：留下明確的下一章動力（新危機、新情報、高風險決定），拒絕平淡收尾。

### Step 3: 自我審查清單 (Self-Correction)
在將內容提交給使用者之前，在內心（thought process）確認以下事項：
- [ ] 是否不小心加入了大段純設定說教？（如果有，立刻改寫為動作與對話）
- [ ] 敵方、系統或世界觀是否被降智了？（壓迫感必須始終存在）
- [ ] 戰鬥是否只展示了力量，而沒有改變核心狀態（資源、情報、權限或生死危機）？
- [ ] 是否忘了寫代價？爽點與收穫是不是來得太容易了？
- [ ] 本章結尾是否足以把讀者「踹進」下一章？
- [ ] **字數檢查：本章正文是否確實超過了 3000 字？**

## 💡 風格指導
- **文字風格**：使用冷峻、節奏緊湊的筆觸，強化物理感、疼痛感與腐朽龐大的賽博/廢土氛圍。
- **人物形塑**：主角在掠食者社會中求生，但仍保留人類記憶的殘響。不寫成聖母，也不寫成廉價的冷血機器。
- **神性與機械的結合**：描繪內部文明與主機運作時，強調「冰冷的數據算力」與「狂熱的信仰神性」之間的反差與融合。
