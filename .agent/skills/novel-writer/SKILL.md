# novel-writer 技能：撰寫/續寫《末日母艦》章節

> 載入本技能後，AI 將依循以下指令完成寫作任務。

---

## 第一部分：專案概述

### 小說基本資訊

- **書名**：《末日母艦：我的體內有個微縮文明》（英文：Mothership: Inner Micro-Civilization）
- **類型**：科幻末世 + 階級壓迫 + 文明主機外掛 + 稅戰經濟爽文
- **當前進度**：第 1–180 章完稿（第六卷第二幕第一個循環完成），下一章接第 181 章，繼續第六卷第二幕「權限戰爭」
- **一句話定位**：母艦 = 封閉作業系統，主角 = 入侵的病毒文明

### 核心隱喻

母艦「方舟」是一個封閉的物理作業系統。氧氣、空間、重力 = 硬體資源；存活點 = 低頻算力；信仰 = 高頻算力。主角林燼體內的「文明主機」是高維度新 OS，寄生在母艦中運行。

## 第三部分：寫作規則濃縮（11 條精華）

詳細規則見 `RULES.md`。以下為續寫前必須默記的核心：

1. **字數**：每章 ≥ 3000 字，常態 3200–4500
2. **節奏**：3 章一循環（行動 → 衝突 → 結算）
3. **視角**：每章鎖定單一感知主體，不跳入其他角色內心
4. **資訊差**：對話中角色不該解釋對方已知的事；保留「角色不說的話」作為張力
5. **禁止**：大綱感、企劃語、AI 味（連續成語、括號標籤、總結句）
6. **反派必備**：先傲慢 → 再錯愕 → 再恐懼（詳見 `RULES.md` §8 反派誤判公式）
7. **主角**：少說話、開掛後不苦戰、每章要有收益
8. **系統提示**：像高維文明介入，非遊戲面板（如【權限判定中】，非【系統公告】）。台詞範本見 `STYLE.md` §5
9. **結尾**：必須留鉤子，不是結論
10. **禁止污染**：不得出現「工作流：」「模式 B」「本章爽點」等流程殘留用語
11. **沉浸**：不得打破第四面牆，禁止後設用語

## 第五部分：品質閘門（寫完一章後強制檢查）

完整檢查表見 `RULES.md` §6。最低限度六關：

1. **視角是否鎖定？** — 全章是否只有一個感知主體？是否跳入了其他角色的內心？
2. **資訊差是否存在？** — 角色對話中有沒有解釋對方已知的事？有沒有保留「角色不說的話」？
3. **感知寫作是否到位？** — 場景是「看到、聽到、聞到、感覺到」還是「說明文」？
4. **場景化是否完成？** — 事件是否發生在具體的物理空間中？還是在真空中推進？
5. **結尾是否有鉤子？** — 最後一段是否留下懸念、轉折或不可逆變化？
6. **字數是否達標？** — ≥ 3000 字？若不達標，是否在 3200–4500 常態區間？

**任一關未過 → 退回修訂，不得視為完稿。**

## 第六部分：續寫工作流

完成一章正文並通過品質閘門後，執行以下後續步驟：

1. **更新 `02_PROJECT_DATABASE/STATE.md`**
   - §1 快照表：更新最新章節、版本、暴露概率、倒數計時器
   - §4 外部時間線：新增剛完成的章節事件
   - §6 開放伏筆：標記回收、新增埋設
   - §7 衝突弧線：更新弧線位置
   - §9 第六卷第二幕決策清單：標記已完成項
2. **更新 `.ai/SUMMARY.md`**
   - 更新進行中的敘事線描述
   - 確認關鍵詞彙表無需增補
3. **【名詞自動收錄】(Term Mining Protocol)**
   - 審視剛寫完的章節，是否有引入新的設定名詞、特殊制度、機構或能力名稱。
   - 若有，**必須自動**將其加入 `02_PROJECT_DATABASE/_GLOSSARY.json`。
   - **格式要求：**
     - `term`: 名詞本身。
     - `internal_note`: 對作者/AI的客觀定義解釋。
     - `display_text`: 對讀者的「感知描寫」（帶有機油味/血腥味的系統提示格式，使用 `【】` 包覆）。
     - `v_version`: 當前演化階段（如 V6）。
     - `source`: 分類（世界 / 角色 / 力量）。
4. **若有新世界/角色/能力細節** → 同步更新對應 `CANON_*.md`（罕見）
